## 1. 调研范围与方法简述

- **代码范围（后端）**：  
  - `UHA/scp-foundation` 下主要模块：`scp-ips-*`、`scp-mds-*`、`scp-sds-*`、`scp-sop-*`、`scp-mps-*`、`scp-mrp-*`、`scp-ams-*`（SCH）、`scp-dfp-*`；  
  - 通用模块：`scp-common`；  
  - 其它具备用户可见错误返回的模块：如算法服务 `scp-das-service` 等。  
- **前端范围说明**：  
  - PRD 要求覆盖 `scp-front/` 下所有子项目，但当前工作空间根目录为 `UHA/scp-foundation`，**未包含 `scp-front` 工程目录**，因此本轮调研仅能覆盖**后端及公共模块**，前端 `this.$message.error` / `message.error` / `ElMessage.error` / `$notify.error` 等调用点暂无法从代码中直接枚举；  
  - 前端错误提示模式可参考上一份代表性调研中的总结，本次全量清单以“后端返回文案 + 业务异常点”为主。  
- **错误类型聚焦**：  
  - 后端业务异常：`throw new BusinessException(...)`、`BaseResponse.error(...)`、`BaseResponse.fail(...)` 等；  
  - 业务语义明显的日志或 remark 字段中出现如“失败 / 错误 / 不存在 / 无效 / 不能为空 / 不允许 / 未配置 / 未维护 / 非法”等文案，且有高概率透传给前端或导出文件；  
  - 技术性异常（SQL/连接错误等）只在其被直接拼接进用户文案时才收录。  
- **检索方法**（按模块分批执行）：  
  - Java 侧：  
    - `BusinessException`、`BaseResponse.error(`、`BaseResponse.fail(`；  
    - 字符串中包含关键词：“失败”、“错误”、“不存在”、“无效”、“不能为空”、“不允许”、“未配置”、“未维护”、“非法”、“报错”等；  
  - 补充按子模块路径扫描（如 `scp-ams-sdk`、`scp-das-service` 等），确保排程、算法等模块的错误点被覆盖。  

> 说明：  
> - 下文表格中的“是否用户可见”为基于代码调用方式的**推断**：如果文案直接出现在 `BaseResponse.error` / `BusinessException` 或导出错误文件 remark 中，标记为“是”；如仅为日志或内部错误信息，标记为“否/不确定”。  
> - 为控制篇幅，部分重复或完全相同的文案按“同类合并”的方式归入一行，并在备注中标记“多处使用”。

## 2. 总体观察（在代表性调研基础上的补充）

在上一份代表性调研的总体观察基础上，本轮“尽可能全量”梳理有以下**新增/强化的共性发现**：

1. **“参数不能为空 / 未配置 / 未维护”类文案在 MDS/SDS/DFP/AMS 等模块中大量重复出现，且多为技术字段名**  
   - 如 `voClassName不能为空`、`dtoClassName不能为空`、`参数不能为空`、`产品类型错误`、`资源规格切换时间不能为空` 等，用户层面对这些字段名难以理解。  
2. **排程与手工排程（AMS/SCH）模块中存在大量细粒度规则报错，但文案多为规则结果本身，缺乏业务背景**  
   - 例如“当前工序不允许拆分，请确认！”、“仅对于未计划数量不为0的订单可选“新增计划””、“无法垮资源类别拖拽”等，规则丰富但缺少“为什么 / 在哪里看状态 / 如何修正”的说明。  
3. **SDS（订单/供需平衡）和 MRP/MPS 模块中，与工艺路径、制造订单拆分/合并相关的错误点较密集**  
   - 包含多种“未匹配到有效工艺路径”“合并失败,未维护制造订单拆分规则”“制造订单计划状态不允许删除”等，体现出当前在“配置依赖 + 状态机控制”上的复杂度。  
4. **全局进度与任务中断（`scp-common` + 各模块 Schedule）存在统一能力，但文案风格不统一**  
   - 如“中断任务失败: xxx”、“当前进度状态为失败或成功,已结束,无法中断”、“中止算法运行失败：xxx”，既有通用控制器，也有各业务模块自己的实现。  
5. **算法调用与结果解析相关的错误在 SOP/MRP/MPS/AMS/DFP/DAS 中多点分散，常直接拼接 `e.getMessage()`**  
   - 导致用户可能看到技术异常或英文堆栈片段；也缺乏统一的“算法运行失败 / 结果解析失败”模式与错误码。  
6. **前端文案协同缺位在“制造订单/工序操作失败”、“导入导出错误”、“排程拖拽失败”等场景尤为明显**  
   - 大量后端文案中提到“请看同步失败原因字段”“行数：XX；制造订单计划状态不允许删除”等信息，如果前端不显式展示这些字段，用户会只看到简单的“失败”提示。
