# 租户管理 ·「租户克隆」现状与问题分析

**文档类型**：产品知识库 / 用户与实施向说明 + 代码与现象分析  
**模块**：IPS（低代码租户管理页 + 场景服务）  
**菜单路径**：系统设置 → 租户管理（页面内表格「操作」列 · **租户克隆**）  
**分析时间**：2026-03-30  
**代码范围**：`scp-foundation` 内 `scp-ips-front`（`src/views/dc/tenant/table.vue`）、`scp-ips-sdk`（`ScenarioController`、`ScenarioServiceImpl`）、`scp-ips-api`（`ScenarioParam`）（只读调研，未改代码）

## 一、产品侧说明（用户与实施视角）

本节面向 **业务管理员、实施顾问、客服/二线支持**，用「能做什么、不能做什么、界面上会看到什么」描述；与实现细节对应关系见后文 **§二～§四**。

### 1.1 功能定位：解决什么问题

| 维度 | 说明 |
|------|------|
| **典型诉求** | 某个**已存在的租户**需要与另一个环境/模板租户在**数据与配置上对齐**，或某模块缺少「主场景」时，希望**参照系统中已有场景**快速拉齐。 |
| **能力本质** | 在**当前选中的这一行租户**上，按该租户已开通的**业务模块**，为各模块指定一个**来源场景**；保存后系统去**创建或调整该租户下的场景数据**（技术上前端弹窗名为「场景克隆」）。 |
| **不解决什么** | **不会**在点击一次后自动生成**新的租户编码**、**新的管理员账号**或**完整复制租户行**；若需要「再建一个租户」，应走 **租户新增/开通** 等流程，再视需要对本功能做场景对齐。 |

### 1.2 界面对象：用户看到的三个层次

为避免培训时混淆，建议统一话术：

1. **租户**：列表中每一行，含租户编码、租户名、所含模块、管理员账号等。用户点的是**这一行**上的「租户克隆」。  
2. **模块**：租户「所含模块」列中的 MDS、IPS、MRP 等；弹窗里会按**该租户已开通的模块**出现一行行「某某模块场景」的下拉（前提是系统里该模块**已有可选场景**，见 **§1.5**）。  
3. **场景**：每个业务模块下的**一套数据环境**（含独立库表空间等业务含义）；下拉框里选的是**用哪个已有场景作为来源**。  

**产品提示**：列表按钮叫「租户克隆」，弹窗标题叫「场景克隆」——对用户容易理解为两件事。对外说明时建议明确：**「在指定租户下，按模块选择来源场景并落库」**，避免口头简称成「复制租户」。

### 1.3 推荐操作路径（给最终用户的步骤说明）

1. 进入 **系统设置 → 租户管理**，在列表中找到**要被调整场景的那一行租户**（目标租户）。  
2. 点击该行 **「租户克隆」**。  
3. 若正常，会弹出 **「场景克隆」** 对话框，内部按模块展示下拉框（例如「MDS 场景：」「MRP 场景：」等，具体以实际开通模块为准）。  
4. 为需要配置的**每一个下拉框**选择**来源场景**（一般选模板租户或标杆租户下已验证的场景）。  
5. 点击 **确定**；成功时通常会有关闭弹窗与成功提示，列表可刷新查看。  

若弹窗**只有标题没有下拉框**，或点击后**长时间无任何弹窗**，不要反复猛点，应按 **§五** 由实施或研发做环境与接口排查（常见为无可用场景数据或请求失败未提示）。

### 1.4 能力边界：能做什么 / 不能做什么

**当前实现下，用户侧可合理预期的范围：**

| 可以做 | 不建议预期 |
|--------|------------|
| 对**目标租户**下、弹窗里出现的模块，选择**来源场景**并保存 | 一键得到**另一个新租户**（新编码、新账号） |
| 某模块**尚未有主场景**时，保存可能触发**新建场景库并复制结构/数据**（实施需结合环境观察耗时与结果） | 所有模块都一定是「完整复制一份库」；**已有主场景**的模块当前更偏向**改来源关系**，与「再拷一份库」不同（见 **§三.2**、**§四.6**） |
| 来源场景列表来自**全系统已维护的场景**，用于对齐与初始化 | 下拉列表为空时仍强行认为「系统坏了」；更可能是**该模块在系统里还没有任何场景**或租户模块未对齐 |

### 1.5 为什么有时「像没反应」——用户可见现象与解释话术

| 用户描述 | 建议对用户的解释方向（不涉及代码细节） |
|----------|----------------------------------------|
| 点了没弹窗 | 可能是网络、权限或系统异常；请换浏览器重试或联系管理员看后台日志；**不要认定按钮无效**。 |
| 弹窗是空的 | 当前租户虽开通了模块，但系统里**这些模块还没有可作为模板的场景数据**，或暂时拉不到列表；需先在对应模块里维护场景，或由实施核对数据。 |
| 选了确定没提示 / 没变化 | 可能未选全必填项、或保存失败但界面未提示；请管理员查接口返回；实施可按 **§五** 排查。 |
| 我以为会复制整个租户 | 本功能是在**当前租户**下配置**各模块场景来源**，不是「再复制一行租户」；新租户请走**新增租户**流程。 |

### 1.6 与相关能力的关系（产品矩阵）

| 能力 | 与「租户克隆」的关系 |
|------|----------------------|
| **租户新增 / 编辑** | 管租户身份、模块开关、管理员等；**不替代**本功能的场景来源配置。 |
| **各模块内场景管理** | 产生可被下拉选中的「来源场景」；无场景则克隆弹窗可能为空。 |
| **权限 / 系统管理员** | 接口侧偏系统管理；若仅业务租户管理员登录，可能因权限看不到数据或无法保存，需与 **SYSTEM_ADMIN** 类角色策略对齐。 |

### 1.7 给实施与培训的「一句话」

> **在租户管理里点「租户克隆」，是在「当前这一租户」上打开「场景克隆」弹窗：按已开通模块选择来源场景并保存；不会自动生成新租户行；弹窗为空多半是模块下还没有可选场景或请求失败。**

## 二、前端实现逻辑（点击后应发生什么）

### 2.1 入口与交互

| 项 | 说明 |
|----|------|
| 页面组件 | `scp-front/scp-ips-front/src/views/dc/tenant/table.vue` |
| 操作列 | 自定义列 `fscope: true`，插槽内 **`el-tag`**，文案「租户克隆」，`@click="handleCopy(scope.row)"` |
| 点击后 | `handleCopy`：`this.tenantId = record.id` → 异步 `getScenario(this.tenantId)` → **同步** `this.copyVisible = true` → `this.ruleForm = {}` |

### 2.2 拉取弹窗数据

- **接口**：`GET`（经 `baseUrl`）**`/scenario/clone/data`**，参数 `tenantId`  
- **封装**：`src/api/dc/tenant.js` → `getScenario`  
- **成功**：`this.scenarioList = res.data.data`（期望为**按模块编码为 key** 的对象，value 为下拉选项数组）  
- **失败**：`else` 分支里使用 `data.data.msg`（见 **§四.1**，存在变量作用域问题）

### 2.3 确认保存

- **接口**：`POST` **`/scenario/save`**，`saveScenario`，请求体 `{ tenantId, param: [{ moduleCode, sourceScenarioId }, ...] }`  
- **param 来源**：`_.map(this.ruleForm, ...)`，即用户在弹窗内各下拉框选中的值  
- **成功**：提示成功、刷新列表、关闭弹窗  
- **失败**：同样存在 **§四.1** 的 `else` 分支问题

## 三、后端实现逻辑

### 3.1 克隆弹窗数据：`GET /scenario/clone/data`

**类**：`com.yhl.scp.ips.system.controller.ScenarioController#getCloneData`（`/scenario/clone/data`）

**逻辑摘要**：

1. 按 `tenantId` 查 **`TenantModule`**（该租户所含模块）。  
2. **`scenarioService.selectAll()`** 取**全库**场景，按 `moduleCode` 分组。  
3. 对每个租户模块：若该模块下存在场景，则组装 `LabelValue` 列表（展示名、场景 id）放入 `result[moduleCode]`。  
4. 若某模块在全局场景中**没有任何记录**，则 **continue**，该模块**不出现在 result 中**。  
5. 返回 `BaseResponse<>(true, result)`。

**含义**：弹窗里**只会出现「租户已开通 + 且全局存在该模块场景」的模块**；若 `result` 为空对象 `{}`，则弹窗表单区域**没有任何下拉框**，用户极易感知为「点了没反应 / 空白」。

### 3.2 保存：`POST /scenario/save`

**类**：`ScenarioController#save` → **`ScenarioServiceImpl#doSave(ScenarioParam)`**

**逻辑摘要**：

1. 查询目标租户下**主场景**（`master = YES`），按 `moduleCode` 建成 `scenarioMap`。  
2. **`param` 为空** → 直接 **`BaseResponse<>(false, "创建失败")`**。  
3. 对每个 `Param`：  
   - **若该模块在目标租户已存在主场景**：仅 **`master.setSourceScenarioId`** + **`updateByPrimaryKey`**，注释 **`// TODO 克隆`** —— **未在此处执行库表复制类克隆**。  
   - **若不存在主场景**：走 **`createScenario`**（建库、复制表/视图等，失败会回滚删除并抛错）。

**含义**：同一租户「已有主场景」的模块，当前保存更像**改来源场景 id**；**完整数据克隆**主要发生在新主场景分支。与用户对「克隆」的理解可能不一致。

## 四、「点击无效果」的可能原因（按优先级归纳）

### 4.1 前端缺陷：失败分支引用未定义变量（高）

在 **`getScenario` 的 `then`** 与 **`handleOk` 的 `then`** 中，`success` 为 `false` 时的 `else` 使用了 **`data.data.msg`**，但 **`data` 仅在 `if` 块内 `let { data } = res.data` 声明**，`else` 中 **`data` 不在作用域**，在严格模式下可导致 **`ReferenceError`**，Promise 链异常，**界面上可能既无错误提示、流程也中断**，表现为「没反应」。

（成功路径不触发该问题。）

### 4.2 弹窗打开但内容为空（高）

- **`/scenario/clone/data` 返回 `{}`**：例如租户模块与全局场景无法匹配、或场景表无数据等 → **`v-for="(value, key) in scenarioList"`** 无项可渲染 → 仅标题「场景克隆」、无表单项。  
- **接口失败进入 `catch`**：当前 **`catch` 内未统一 `Message` 提示**（部分被注释），**`scenarioList` 可能仍为上一次打开的旧值或初始值**，表现混乱或仍像「无效」。

### 4.3 权限 / 网关 / 环境（中）

- 接口注释含 **SYSTEM_ADMIN** 语义；若网关或权限拦截 **`/scenario/clone/data`** 或 **`/scenario/save`**，需看实际返回是否为 401/403 及前端拦截器是否仅跳转登录而无业务提示。  
- **`baseUrl`** 指向错误环境时，请求失败同样可能落入静默 `catch`。

### 4.4 表格内点击被组件吞掉（中·需现场验证）

「租户克隆」在 **`yhl-table` 自定义列**内，若表格对行/单元格做了 **`stopPropagation` 或遮罩**，存在 **`click` 未触达 `el-tag` 的理论可能**。建议在浏览器开发者工具中确认 **`handleCopy` 是否被调用**（断点或临时 `console.log`）。

### 4.5 业务数据导致「保存」无正向结果（中）

- 用户未选满下拉即确定 → `ruleForm` 映射出的 **`param` 含空 `sourceScenarioId`** 或 **`param` 为空数组** → 后端 **`创建失败`**；若同时触发 **§4.1**，前端仍可能无明确错误提示。

### 4.6 后端「覆盖主场景」与「克隆」预期不符（低·产品认知）

对已存在主场景的模块，当前实现以 **更新 `sourceScenarioId` + TODO** 为主，**不等于**再次执行完整库复制。用户若对比「新模块建场景」与「已有模块」行为，可能认为「克隆未生效」。

## 五、建议排查顺序（实施 / 测试）

1. 打开浏览器 **Network**：点击「租户克隆」后是否出现 **`/scenario/clone/data?tenantId=...`**，状态码与响应体 `data` 是否为空对象。  
2. 打开 **Console**：是否有 **`ReferenceError: data is not defined`** 或其它未捕获异常。  
3. 确认当前账号对 IPS 场景接口的 **权限** 与 **网关路由**。  
4. 对照 **§三.1**，核对库中 **`scenario` 与租户模块** 是否具备下拉所需数据。  
5. 若需产品化修复：前端应修正 **`else` 分支**（使用 `res.data.msg` 等安全字段）、失败与 **`catch` 统一提示**；产品需界定 **「场景克隆」与「租户克隆」** 文案与能力范围（可与 **§一.2、§一.7** 对外话术对齐），并评估 **主场景已存在时** 是否应执行真实克隆（补齐 `TODO` 或调整交互）。
