# 租户管理 · 租户克隆（场景克隆）功能说明

| 项 | 内容 |
|---|------|
| **文档性质** | 产品功能说明；附录含实现要点与已知问题 |
| **模块** | IPS（租户管理页 + 场景服务） |
| **菜单路径** | 系统设置 → 租户管理 → 操作列 **租户克隆**（弹窗标题 **场景克隆**） |
| **维护说明** | 实现依据：`scp-ips-front`（`table.vue`、`api/dc/tenant.js`）、`ScenarioController`、`ScenarioServiceImpl`、`ScenarioParam` |

---

## 1. 功能概述

在**指定租户**（列表中选中的租户行）上，按该租户**已开通的业务模块**，为各模块选择**来源场景**并保存；系统据此**创建或调整**该租户在各模块下的**主场景**及相关数据环境。

列表入口名「租户克隆」与弹窗名「场景克隆」指同一能力。对外宜表述为：**按模块为租户配置场景来源并落库**，避免简称为「复制租户」。

---

## 2. 术语

| 术语 | 说明 |
|------|------|
| **租户** | 租户管理列表中的一行，表示独立客户空间（含编码、名称、开通模块、管理员等展示信息）。 |
| **账号** | 登录主体，归属租户；**本功能不创建新租户行、不自动生成新管理员账号**。 |
| **模块** | 如 MDS、S&OP、MRP 等业务子系统；以租户模块配置为准。 |
| **场景** | 某模块下的数据运行环境，通常对应独立库或独立数据空间；来源场景可作模板。 |
| **主场景** | 某租户在某模块下的主用场景（`master = YES`）；保存逻辑以其是否存在为分支条件。 |

---

## 3. 功能范围

**范围内**：为**当前选中租户**按模块指定来源场景；无主场景时新建场景库并从来源复制表与视图等；有主场景时更新来源场景引用（详见 §6）。

**范围外**：不新增租户记录、不修改租户编码、不替代租户新增/开通及租户编辑中的身份与账号维护。

---

## 4. 操作步骤

1. 进入 **系统设置 → 租户管理**。  
2. 选中需配置的**目标租户**行。  
3. 点击 **租户克隆**，打开 **场景克隆** 弹窗。  
4. 在展示的各模块下拉中选择**来源场景**（可选全库中已存在的场景，多用于模板或标杆环境）。  
5. 点击 **确定**；成功则关闭弹窗并提示，列表可刷新查看。

---

## 5. 弹窗数据规则

调用 **`GET /scenario/clone/data?tenantId={目标租户id}`** 加载弹窗。

- **出现某模块下拉的条件**：目标租户**已开通**该模块，且全库**该模块编码（moduleCode）下至少存在一条**场景记录；否则该模块**不出现**（非空行占位）。  
- **下拉选项内容**：全库该 **moduleCode** 下的**全部场景**，**不按目标租户过滤**；与业务用户在模块内日常使用中「仅本租户场景」的列表策略不同。  
- **仅弹窗标题、无表单项**：常为上述条件与全库场景数据的**交集为空**，或接口失败；需结合环境与权限排查。

---

## 6. 保存后业务行为

提交 **`POST /scenario/save`**，请求体含目标 `tenantId` 及各模块的 `moduleCode`、`sourceScenarioId`。后端**按模块分别**处理：

| 条件 | 行为（当前实现） |
|------|------------------|
| 目标租户在该模块**无主场景** | 新建场景记录与独立库，从来源场景对应库**复制表结构、数据及视图**；目标与来源**使用不同库**，复制为保存时刻的快照，后续变更不同步。 |
| 目标租户在该模块**已有主场景** | 以更新主场景的**来源场景标识**为主；**不切换**目标已有主场景绑定的库；完整再次克隆未在分支内实现（代码留 `// TODO 克隆`）。 |

各模块保存结果相互独立；未完整选择或参数异常可能导致保存失败。

---

## 7. 权限与数据可见性

克隆数据与保存接口面向**系统管理**场景（注释含 SYSTEM_ADMIN 语义）。业务租户管理员若权限不足，可能出现无数据或保存失败，需与角色与网关策略一致。

---

## 8. 与其他功能的关系

| 功能 | 关系 |
|------|------|
| **租户新增 / 编辑** | 维护租户身份、模块开关、管理员等；不能替代本功能的场景来源配置。 |
| **各模块内场景管理** | 维护全库场景数据；无场景则弹窗可能无对应模块或内容为空。 |

---

## 9. 异常与用户沟通要点

| 现象 | 处理与说明方向 |
|------|----------------|
| 点击无弹窗或长时间无响应 | 查网络、权限、网关；勿重复猛点。 |
| 弹窗无表单项 | 多为 §5 数据条件不满足或接口失败。 |
| 用户认为会复制整租户 | 说明仅配置**当前租户**下各模块场景，新租户走**租户新增**。 |
| 确定后无明确提示 | 查接口返回；参见附录已知前端缺陷。 |

## 附录 A · 实现与已知问题（研发 / 测试）

- **前端**：`table.vue` 中 `handleCopy` → `getScenario`；保存 `handleOk` → `POST /scenario/save`。  
- **`getCloneData`**：`TenantModule` 与 `scenarioService.selectAll()` 按 `moduleCode` 分组结果求交并组装下拉。  
- **`doSave`**：按主场景存在性分支至 `updateByPrimaryKey`（更新 `sourceScenarioId`）或 `createScenario`（建库 + `doTablesCopy` / `doViewsCopy`）。  
- **已知缺陷**：`getScenario`、`handleOk` 成功回调的 `else` 分支使用 `data.data.msg`，但 `data` 仅在 `if` 内声明，严格模式下可触发 **`ReferenceError: data is not defined`**，导致失败无提示；`catch` 提示不完整。修复方向：统一使用安全字段（如 `res.data.msg`）并完善 `catch` 提示。

**代码路径**：`scp-ips-front/src/views/dc/tenant/table.vue`、`src/api/dc/tenant.js`、`ScenarioController.java`、`ScenarioServiceImpl.java`、`ScenarioParam.java`。
