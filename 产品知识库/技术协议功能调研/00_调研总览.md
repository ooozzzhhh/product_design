## 二、产品模块结构

### 2.1 已识别的核心模块

| 模块代码 | 模块名称 | 数据库 | 说明 |
|---------|---------|--------|------|
| IPS | 集成平台服务 | scp_ips | 系统管理、权限、租户、流程引擎、任务调度 |
| SOP | 产销协同计划 | scp_sop | S&OP 计划、供需平衡、KPI |
| MDS | 主数据服务 | scp_mds | 物料、BOM、工艺路线、日历、供应商 |
| MPS | 主生产计划 | scp_mps | 主生产排程、工厂分配 |
| MRP | 物料需求计划 | scp_mrp | 物料需求计算 |
| DFP | 需求预测 | scp_dfp | 需求预测算法、预测结果管理 |
| AMS | 高级排程 | scp_ams | 生产排程（原 SCH） |
| SDS | 供需决策 | scp_sds | 供需平衡、库存、订单履行 |
| ODS | 订单需求服务 | scp_ods | 订单管理 |
| CTS | 协同任务服务 | scp_cts | 任务协同 |
| DAS | 分布式算法服务 | scp-das-service | 算法执行器 |
| DCP | 数据集成平台 | scp-dcp-service | 数据集成 |

### 2.2 技术栈

- **后端框架**:Spring Boot + Spring Cloud
- **流程引擎**:Flowable (BPMN 2.0)
- **任务调度**:XXL-Job
- **数据库**:MySQL
- **缓存**:Redis
- **消息队列**:Redis Stream
- **API 网关**:scp-gateway-service

## 三、高风险功能初步评估

基于代码搜索和数据库结构,以下功能存在**高风险**（可能超出产品能力或表述过于宽泛）:

### 3.1 需求预测算法

**技术协议承诺**:「搭建机器学习、时序分析、多变量回归等多种融合算法」

**实际情况**:
- ✅ 存在 DFP 模块（需求预测）
- ✅ 存在预测结果管理、版本管理、多维对比功能
- ⚠️ **未找到机器学习框架集成**（如 TensorFlow、PyTorch、Scikit-learn）
- ⚠️ **未找到时序分析库**（如 Prophet、ARIMA）
- ⚠️ **未找到多变量回归实现**

**风险等级**:🔴 高风险
**建议**:改为「支持多种预测算法模板,可扩展集成第三方预测引擎」

### 3.2 流程编排插件化二开

**技术协议承诺**:「流程编排插件化二开支持对流程编排中的插件进行二开」

**实际情况**:
- ✅ 集成 Flowable 流程引擎
- ✅ 支持 BPMN 2.0 流程建模
- ✅ 存在流程模型管理、流程实例监控
- ⚠️ **插件化二开能力有限**:Flowable 支持 Java Delegate 扩展,但非「插件市场」模式

**风险等级**:🟡 中风险
**建议**:改为「支持通过 Java Delegate 扩展流程节点,实现自定义业务逻辑」

### 3.3 多租户架构

**技术协议承诺**:「支持多租户架构,实现不同租户之间的数据、配置、功能权限的严格隔离」

**实际情况**:
- ✅ 存在租户表 `auth_rbac_tenant`
- ✅ 存在租户-用户关联表 `auth_rbac_tenant_user`
- ✅ 存在租户-模块关联表 `auth_rbac_tenant_module`
- ✅ 存在数据权限配置表 `auth_rbac_data_permission_config`
- ✅ 存在大量 `scp_sys_xxx_<随机码>` 数据库（疑似租户隔离数据库）
- ⚠️ **数据隔离方式**:采用「租户独立数据库」模式（每个租户一个独立数据库）

**风险等级**:🟢 低风险
**建议**:保留,但需明确「采用租户独立数据库模式实现数据隔离」

### 3.4 算法模板可插拔

**技术协议承诺**:「支持二开算法模型接入」

**实际情况**:
- ✅ 存在算法服务模块 DAS (scp-das-service)
- ✅ 存在算法输入/输出服务
- ⚠️ **算法模板管理**:未找到独立的「算法模板库」或「算法插件注册机制」
- ⚠️ **算法扩展方式**:需通过代码开发 + 部署,非「热插拔」

**风险等级**:🟡 中风险
**建议**:改为「支持通过配置化方式管理算法工作流,可扩展集成自定义算法服务」

### 3.5 报表设计功能

**技术协议承诺**:「提供强大的报表设计功能来满足用户设计新报表或修改已有报表的需求」

**实际情况**:
- ❌ **未找到报表设计器**（如 UReport、积木报表、JasperReports）
- ✅ 存在大量业务报表 Controller（如 `ProductionSalesInventoryController`、`HistoricalInventoryLevelsController`）
- ⚠️ **报表实现方式**:通过后端 API + 前端页面硬编码实现,非「可视化报表设计」

**风险等级**:🔴 高风险
**建议**:改为「支持定制化报表开发,约定 10 张以内报表」,删除「强大的报表设计功能」表述

## 四、调研结论摘要

### 4.1 可以保留的功能（产品已实现）

1. ✅ 多租户管理（租户独立数据库模式）
2. ✅ 基础权限管理（菜单、角色、用户、数据权限）
3. ✅ 流程引擎（Flowable BPMN 2.0）
4. ✅ 任务调度（XXL-Job）
5. ✅ 数据集成管理（数据源、数据集、连接定义）
6. ✅ S&OP 计划管理（需求预测、供需平衡、存货管理）
7. ✅ 主数据管理（物料、BOM、工艺路线、日历）
8. ✅ 计划算法管理（算法数据源、算法工作流）
9. ✅ KPI 看板（已有 KPI 相关 Controller 和 Service）
10. ✅ 日志管理（操作日志、系统日志、Excel 导入日志）

### 4.2 需要调整表述的功能（部分实现或表述过于宽泛）

1. 🟡 需求预测算法:删除「机器学习、时序分析、多变量回归」,改为「支持多种预测算法模板」
2. 🟡 流程编排插件化二开:改为「支持通过 Java Delegate 扩展流程节点」
3. 🟡 算法模板可插拔:改为「支持配置化算法工作流管理」
4. 🟡 报表设计功能:删除「强大的报表设计功能」,改为「支持定制化报表开发」
5. 🟡 KPI 运营看板:删除「自定义业务规则」,改为「支持配置化 KPI 指标展示」

### 4.3 需要删除或标注为「可扩展」的功能（未实现）

1. ❌ 机器学习预测算法
2. ❌ 可视化报表设计器
3. ❌ 插件市场模式的流程编排二开
