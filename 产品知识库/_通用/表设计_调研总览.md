## 2. 跨库逻辑关联索引

MySQL 侧 **通常不为跨库表建立外键**；下列为理解**多模块库与 IPS 场景**时的常用逻辑关联（非穷举）。

| 关系 | 说明 | 依据 |
|------|------|------|
| IPS 场景 → 业务库 | `scp_ips` 中 `auth_scenario` 等维护场景与 **数据源库名、连接**，服务可切换至 `scp_sds`、`scp_ams`、`scp_mps`、`scp_sop`、`scp_dfp` 等。 | `scp_ips.auth_*`；各服务动态数据源（代码侧） |
| IPS 用户 → 场景 | `auth_rbac_user_scenario` 将用户授权到场景，决定可访问的业务库范围。 | `scp_ips.auth_rbac_user_scenario` |
| 多业务库「同名前缀」 | `scp_sds`、`scp_ams`、`scp_dfp`、`scp_mps`、`scp_sop` 等常同时含 **`mds_*`、`dps_*`、`sds_*`** 等命名空间，**结构相似但 schema 不同**；数据是否一致取决于场景与同步，**不能仅凭表名推断行级相同**。 | 各模块 `01_表与视图清单` 前缀统计对比 |
| SDS 文档引用 `scp_mps` | SDS 相关代码调研为核对因子等会引用 **`scp_mps`** 表/视图。 | [SDS/代码/SDS_需求优先级编码方案字段建议复核_代码实现.md](./SDS/代码/SDS_需求优先级编码方案字段建议复核_代码实现.md) |
| SCH 客户订单全链 | 客户订单多跳关联以 SCH 归档为准。 | [SCH/客户订单关联业务数据表分析.md](./SCH/客户订单关联业务数据表分析.md) |
| SOP 能力边界 | 协议层功能描述可与表设计对照。 | [技术协议功能调研/03_SOP计划与报表管理.md](./技术协议功能调研/03_SOP计划与报表管理.md) |
