# SCH 模块 — 数据源与调研说明

## 环境与范围

| 项 | 说明 |
|----|------|
| **环境** | 测试环境 MySQL，`10.7.60.157:3306` |
| **逻辑库** | `scp_ams`（产品侧常称 AMS；知识库目录沿用 **SCH**） |
| **产品模块** | 高级排程 / 生产排程 |
| **调研日期** | 2026-03-24 |

## 库形态说明

- **典型合一库**：`scp_ams` 内同时包含 **`mds_*`、`dps_*`、`sds_*`** 以及少量 **`ams_*`、`ods_*`、`mrp_*`** 等前缀表，与「仅排程」的狭义模块边界不完全一一对应；阅读表设计时需**按表名前缀区分业务域**。
- **与 `scp_sds` 的对比**：测试环境中 `scp_ams` 与 `scp_sds` 在表命名体系上相似（均大量含 `mds_*` / `dps_*` / `sds_*`），但为**不同 schema**，应用通过 **IPS 场景（`auth_scenario` 等）** 切换数据源，而非跨库外键。跨库说明见 [表设计_调研总览.md](../../表设计_调研总览.md)。
- **既有业务梳理**：客户订单多跳关联、供需 pegging、工单与工序等，详见同目录上级文档 [客户订单关联业务数据表分析.md](../客户订单关联业务数据表分析.md)（已按 `scp_ams` 核对）。
