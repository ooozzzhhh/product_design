# 汇川 POC — R1_MPS 按比例分工厂「增补变更设计」

## 文档说明

| 项目 | 内容 |
|------|------|
| 关联基线 | `R1_MPS按比例分工厂_设计方案.md`；实现分支参考 release17（`FactoryProportionRule`、`mps_fac_factory_proportion_config` 等） |
| 增补性质 | 在**多工厂按比例分配**能力上，除既有**优先级**外，增加**期间维度**与**拆单相关**可配置项；本文件仅描述增量，不重复基线全文。**各期间之间不做结转、不跨期传递缺口或结余。** |
| 读者 | 产品、实施、开发、测试 |

## 1. 增补目标与范围

### 1.1 目标

1. **按期间独立计算分配**：每个期间内单独聚合需求、单独计算「目标量—分配量」并执行趋近分配；**下一期间不读取、不继承、不调整**上一期间的任何分配偏差。
2. **拆单策略可配置**：是否允许拆单、拆单粒度（最小拆单单位）、拆单门槛（最小拆单批量）。

### 1.2 非目标（本轮明确不做或另立项）

- **期间结转 / 跨期联动**：不在本方案范围内；若未来需要「上期少分下期补课」等能力，另立需求与设计。
- 不改变下游 MRP 对 `sds_dem_factory_bom_result` 的消费契约的**结论**：仍为多行结果表达多工厂/多片段数量即可；若拆单导致 `1 需求 → N 行`，需在实现阶段与 SDS/MRP 接口字段做一次兼容性核对（本设计给出数据与追溯要求，不展开 MRP 改造方案）。
- 不替代 APS 细排程的「工序级拆批」；此处「拆单」指 **MPS 分工厂阶段**对同一笔需求数量的**工厂间拆分**（或同工厂多产线，若未来扩展）。

## 2. 业务概念与术语

| 术语 | 含义 |
|------|------|
| 期间 | 用户选定的**时间桶**（如自然周、自然月、或「计划期间」编码对应的起止日）。同一需求行按其**参与分配的基准日期**落入唯一期间（见 3.2）。 |
| 期间内目标量 | 该期间内、某物料下，各工厂（+产线）按配置比例计算得到的**应分配数量**（可为小数精度，落库前再按系统舍入规则处理）。 |
| 期间内实分量 | 该期间内算法实际写入分工厂结果的数量合计。 |
| 拆单 | 将**同一笔需求**的可分配数量拆成**多段**，分配到**不同工厂/产线**（或同目标多行合并策略下的多行）。 |
| 最小拆单单位 | 拆分时每一段数量必须是该单位的**整数倍**（例如单位为 1 时，不允许出现 499.9 / 500.1 这类片段）。 |
| 最小拆单批量 | 仅当**订单（需求行）可分配总量**严格大于该阈值时，才允许对该行执行拆单；否则**必须整单**落到单一工厂/产线。 |

## 3. 期间维度设计

### 3.1 配置项（建议粒度：物料级或「物料 + 分工厂方案」级）

> 具体落表可与现有 `mps_fac_factory_proportion_config` 同表扩展字段，或新增「头表 + 明细」；以下为**逻辑配置项**，表结构在详细设计/PRD 阶段定稿。

| 配置项 | 类型 | 说明 |
|--------|------|------|
| 是否启用按期间分配 | 布尔 | 关闭时行为与当前 release17 一致：不按期间分桶，目标量基于**全量待分配集合**一次计算。 |
| 期间类型 | 枚举 | 如：`CALENDAR_WEEK` / `CALENDAR_MONTH` / `MPS_BUCKET`（与主计划期间主数据对齐）等。 |
| 期间边界时区/日历 | 引用 | 与系统工厂日历或租户默认日历一致，避免跨周歧义。 |
| 需求归属期间的基准日期 | 枚举 | 建议默认 **`需求交期（due date）`**；可选「计划开工日」「录入日」等（若选开工日需定义与 MPS 运算顺序依赖，实施时评估）。 |

### 3.2 单期间内的计算对象

对每个「物料 + 比例配置集合」在期间 \(P\) 内：

1. **本期参与分配的需求集合**：基准日期落入 \(P\) 且仍由本规则处理、且可分配量大于 0 的需求行。
2. **本期有效总需求**：上述行的可分配数量之和，记为 \(Q_P\)。
3. **本期各工厂目标量**：\(T_{f,P} = Q_P \times proportion_f\)（与现算法一致，先按优先级组内比例和为 100% 校验）。
4. **本期分配过程**：仅在 \(P\) 内执行带优先级、缺口的分配（见第 4 节与拆单约束结合）；**不读取其它期间**的已分量或偏差。

**说明**：若某行需求跨期间业务含义要强（例如长单跨月），本轮建议仍**整行归属一个期间**（按基准日期），不在本方案引入「单行拆到多期间」；若未来需要，可另开需求定义「跨期拆分规则」。

### 3.3 与现算法顺序的衔接

- 期间内需求排序：在现有「交期 → id」基础上，可增加「同一交期内按数量降序」等**可配置**项；默认保持与线上一致，减少行为突变。
- **第一笔需求特殊处理**（当前代码对首笔按最小优先级选厂）：建议改为「**期间内**首笔」重置，即**每个新期间**重新应用首笔规则，避免不同期间之间的排序上下文互相影响（与是否结转无关，属于期间分桶后的算法边界）。

## 4. 拆单相关配置

### 4.0 配置挂靠对象（显式约定）

| 对象 | 是否挂靠 | 说明 |
|------|----------|------|
| **物品（物料/产品）** | **是** | 「允许拆单、最小拆单单位、最小拆单批量」与**期间相关开关**均与现有「工厂比例分配配置」一致，按 **物料（product）维度** 维护（与 `mps_fac_factory_proportion_config` 同物料上下文，可同表扩展字段或头表+明细，实现阶段定稿）。 |
| **客户订单** | **否** | 不按客户订单号、不按单笔销售订单单独维护拆单策略。运行时分工厂需求行通过 **productId / 物料编码** 匹配到上述物料级配置，再结合该行**自身数量**判断是否满足最小拆单批量等约束。 |

若未来存在「同一物料、不同客户/订单类型要求不同拆单规则」的场景，可作为**扩展**：订单类型或客户级覆盖物料默认值；**本 POC 增补范围不包含**该覆盖层。

### 4.1 配置项

| 配置项 | 类型 | 说明 |
|--------|------|------|
| 允许拆单 | 布尔 | `false`：任一行可分配量必须**一次性**分配至单一工厂/产线（与早期 POC 文案「整单」一致）。`true`：允许生成多段 `FactoryBomResult`。 |
| 最小拆单单位 | 正数（Decimal） | 记为 \(U\)。任意拆出的片段量 \(q\) 须满足：\(q = k \times U\)，\(k\) 为非负整数；未拆分的整单量**不要求**为 \(U\) 的倍数（避免排斥合法小订单）。 |
| 最小拆单批量 | 正数（Decimal） | 记为 \(B\)。仅当该行**可分配总量** \(Q_{line} > B\) 时允许拆单；若 \(Q_{line} \leq B\)，**禁止拆单**（整单到一个厂）。 |

### 4.2 约束关系（必须同时满足）

对某需求行在允许拆单时，拆分为 \(q_1,q_2,\ldots,q_n\)（\(n\geq1\)）：

1. \(\sum q_i = Q_{line}\)（守恒）。
2. 若 \(n>1\)：必须 \(Q_{line} > B\)。
3. 若 \(n>1\)：对每个 \(q_i>0\)，必须 \(q_i = k_i \times U\)。

**推论**：

- 若 \(U>0\) 且 \(Q_{line}\) 不是 \(U\) 的整数倍，则**要么不拆**（整单 \(Q_{line}\) 到一个厂），**要么拆后至多一段可为「余数」**——与第 3 条冲突。因此需**二选一产品规则**（建议写死为下列之一并在界面提示）：

  **规则 R-A（推荐）**：若 \(Q_{line}\) 不是 \(U\) 的整数倍，则**不允许拆单**，整单分配。

  **规则 R-B**：允许拆单时，要求 \(Q_{line}\) 本身为 \(U\) 的整数倍；否则保存配置或分配前校验报错（错误码另配，如 `MPS052`：物料「x」需求数量不是最小拆单单位的整数倍，请修正需求或调整单位配置）。

本方案推荐 **R-A**（对用户友好，不卡数据录入）；若行业强制「全量按托盘整数倍」，再选 **R-B**。

### 4.3 与比例算法的结合方式（行为描述）

在 `允许拆单=false` 时：保持现有「每次选厂，将本行可分配量一次性记入该厂」的逻辑。

在 `允许拆单=true` 时：

1. 在**当前期间**内，按 \(Q_P\) 与比例得到各厂**目标**与**当前实分**（`currentQty` 从 0 开始累计，仅含本期内已决策的分配）。
2. 对当前待分配行，在**满足 4.2** 的前提下，可将 \(Q_{line}\) 拆为若干段依次或并行决策；每段仍走「缺口最大优先（同优先级组内）」。
3. **最小拆单单位**在实现上可转化为：所有**拆出片段**（不含「整单未拆」情形）必须为 \(U\) 的倍数；整单不拆时不受 \(U\) 整除限制（在 R-A 下）。

### 4.4 拆单对追溯与展示的要求

- 同一 `orderId`（需求行）下多行 `FactoryBomResult`：需能区分**片段序号**或**稳定行键**（若表结构已有则复用），以便工作台展示「一行需求拆为多行结果」。
- 日志与排障：记录本行是否拆单、\(U,B\) 取值、是否按期间分桶，便于 POC 验收对比。

## 5. 前端与校验

### 5.1 配置界面

在「工厂比例分配配置」同一物料维度或独立「高级策略」折叠区增加：

- 期间：启用开关、期间类型、基准日期字段说明（文案提示与 MRP 关系）；**不提供**任何「结转 / 跨期继承」类选项。
- 允许拆单、最小拆单单位、最小拆单批量：联动禁用（不允许拆单时，后两项灰显）。
- 保存校验：`U>0`、`B≥0`；若允许拆单且选 R-B，则增加倍数校验规则。

### 5.2 后端校验

- 比例和 100%、优先级唯一等**既有规则不变**。
- 新增：期间类型与日历引用完整性；拆单参数与 R-A/R-B 一致性。

## 6. 测试关注点（摘要）

| 场景 | 期望 |
|------|------|
| 不启用期间 | 与线上一致，不按期间分桶。 |
| 启用期间 | 各期间目标量仅由**该期内**需求合计 \(Q_P\) 决定；**不依赖**其它期间的分配结果。 |
| 不允许拆单 | 每行最多一个 INSERT（或 UPDATE 合并同厂同资源），与现网类似。 |
| 允许拆单 + \(Q\leq B\) | 永不拆，即使缺口很大。 |
| 允许拆单 + \(Q>B\) + R-A + \(U=1\) | 数量 1000 可拆成多个整数段；不可出现非整数段。 |
| 期间边界 | 交期落在边界两侧的两行分属两期，首笔规则各期独立。 |

## 7. 开放决策（供评审拍板）

1. **R-A 与 R-B** 选定默认行业规则。
2. 拆单后 MRP 侧是否需展示「父需求—子分配行」映射（若已有字段则填规范）。
