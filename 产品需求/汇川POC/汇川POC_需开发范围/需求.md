# 汇川 POC — 需开发范围（产品需求）

> 来源：`汇川POC需求说明--需开发部分.md`（业务方 POC 说明转化）  
> 说明：本文以文字描述需求、规则与**数据推演**；不包含流程图。

---

## 1. 需求背景

汇川在多工厂协同与计划排产场景下，希望在现有供应链计划系统能力上进行 POC 验证，核心包括：

- 按**预设比例**将同一产品的需求**整单**分配到多个工厂（在现有规则式分工厂基础上增强，**不可拆分单行需求**）。
- 按**物料维度**配置是否在 MRP 中**冲减库存**，以及对需冲减的原材料按**产业目录（POC 中与物品系列一一对应）**做**比例供给**。

## 3. R1 — 产品按比例分配给多工厂生产

### 3.1 业务目标

在多工厂协同场景下，同一产品可由多个工厂同时生产。需在**人工预设比例**的前提下，将产品的多笔需求订单（需求行）整单分配到各工厂，使得各工厂分配到的**累计数量占比**尽量趋近于预设比例（POC 场景下**物料—工厂—产线为 1:1:1**，与当前系统分到产线颗粒度兼容）。**注意：不允许拆分单笔需求行，必须整单分配**。

### 3.2 与现有能力的关系

- 现有**规则式分工厂**已支持将同一产品不同需求分到不同工厂。
- 本需求在规则或结果层面增加**「比例分配」**相关配置与计算，使分配结果符合预设比例（而非仅依赖规则优先级匹配单工厂/产线）。

### 3.3 功能要求

1. **配置能力**  
   - 支持按**物料（产品）维度**，配置该产品在**各工厂**（及 POC 场景下对应该工厂的产线）的**分配比例**。  
   - 比例字段含义需可解释、可校验（例如：同一物料下各工厂比例之和是否为 100% 或是否允许归一化，需在详细方案中约定，POC 至少支持一种明确算法）。

2. **工作台操作**  
   - 用户在**分工厂工作台**执行分工厂（自动或触发计算类操作，与现有「分工厂」按钮流程对齐）。  
   - 在界面**下方结果区域**展示：同一产品相关的多笔需求行，经过**整单分配**后的**各工厂（产线）归属情况**。

3. **数据与计算规则（文字描述）**  
   - **输入**：物料在各工厂（及产线）的分配比例；需求汇总中的需求数量（及与现有分工厂一致的上下文字段，如订单、交期等，以保持 POC 可演示）。  
   - **输出**：与现有「分工厂结果」结构兼容的结果集，每条需求行分配到唯一的**目标工厂（产线）**。  
   - 算法需保证**确定性**（相同输入得到相同输出），在多笔需求整单分配过程中，使得各工厂累计获得的总数量尽可能趋近于配置的比例。

### 3.4 验收要点（摘要）

- 配置比例后，在工作台执行分工厂，分配后的各工厂总量占比尽可能与预设比例趋近。  
- POC 1:1:1 场景下可完整演示从配置到结果列表。

### 3.5 数据推演示例（R1）

**目的**：用一组假数据说明从「配置 + 需求」到「分工厂结果」的数据变化（实现细节以 PRD/设计为准）。

| 步骤 | 数据对象 | 内容（示例值） |
|------|----------|----------------|
| ① 配置 | 物料 M 的工厂比例 | 工厂 F1 : 工厂 F2 = **60% : 40%**（POC 下 F1、F2 各绑定唯一产线 L1、L2） |
| ② 输入 | 需求汇总行 | 物料 M，两笔需求单，需求1数量 **600**，需求2数量 **400** |
| ③ 处理 | 整单分配 | 累计总需求=1000。按比例应分：F1得600，F2得400。算法依次分配：需求1(600) 整单分给 F1/L1；需求2(400) 整单分给 F2/L2（不拆分单子）。 |
| ④ 输出 | 分工厂结果表 | 两行结果：一行 orderId=M_1、qty=**600**、工厂 F1、产线 L1；另一行 orderId=M_2、qty=**400**、工厂 F2、产线 L2 |

**数据流文字描述**：主数据中的「比例配置」与需求汇总的「多笔需求单」进入分工厂计算服务 → 经过整单分配算法 → 输出为多行「分工厂结果」（每行需求绑定唯一工厂/产线），供界面下表展示及后续 MRP 等模块消费。

## 4. R2 — 库存冲减配置

**实现调研（与 R1 知识库文档同级）**：`产品设计/产品知识库/MRP/代码/MRP_汇川R2_库存冲减与产业目录比例供给_现状与改造点调研.md`（含与 `mrpRun` 接口、净需求链路、改造切入点对照）。

### 4.1 业务目标

按**物料编码维度**配置：由产品需求按 BOM 展开得到的**原材料需求**，是否参与**库存冲减**。  

- **不冲减**：仅按 BOM 计算物料需求即可（不要求用现有库存满足）。  
- **冲减**：在 BOM 展开基础上，用**原材料库存**满足需求，且对同一原材料被**多个产业目录（POC 中对应物品系列，且产品与系列 1:1）**占用时，按**人工设定比例**将可用库存分配给各产业目录下的产品需求。

### 4.2 功能要求

1. **物料级冲减开关**  
   - 支持按**物料编码**配置：该原材料展开需求是否执行库存冲减。

2. **产业目录与比例**  
   - 支持维护**原材料**向不同**产业目录**供给的比例（POC：产业目录与**物品系列**一致；产品与系列 1:1，多产品可共用同一种原材料）。  
   - 比例用于：在「冲减」模式下，将原材料可分配库存按比例分摊到各产业目录对应的需求上。

3. **MRP 运行**  
   - 用户配置完成后，通过**运行 MRP** 得到物料需求相关结果；结果需能区分「冲减 / 不冲减」行为差异。

### 4.3 期望结果（与 POC 说明对齐的业务结论）

- **产品 A1** 相关需求：配置为**不进行库存冲减** → MRP 输出中表现为按 BOM 计算的物料需求，**不因**原材料库存而减少净需求或特殊分摊。  
- **产品 E9**（归属产业目录 **A** / 物品系列 A）、**产品 E10**（归属产业目录 **B** / 物品系列 B）：对共用原材料 **Y00015**，配置冲减且按产业目录比例分配库存后：  
  - E9、E10 侧展示为：**无额外未满足物料需求**（或净需求为 0，以系统字段定义为准），库存侧体现 **Y00015** 在 A、B 两个维度下分别剩余 **700** 与 **300**（与 POC 给定示例一致，具体字段名以实现为准）。

### 4.4 验收要点（摘要）

- 同一原材料、多产业目录时，冲减后各目录分摊量与配置比例一致（含舍入约定）。  
- 关闭冲减时，不出现「按库存扣减净需求」的 POC 期望行为。

### 4.5 数据推演示例（R2）

**目的**：用一组假数据串起「BOM 展开 → 冲减开关 → 产业目录比例 → MRP 输出与库存视图」的数据流（分配算法细节以实现为准，此处仅表达业务可读推演）。

**主数据关系（POC 假设）**

- 产品 **A1**：归属系列 SA（示例），其 BOM 用到原材料 **Y00015**（数量示例略）。  
- 产品 **E9**：归属产业目录 **A**（物品系列 A）。  
- 产品 **E10**：归属产业目录 **B**（物品系列 B）。  
- **E9、E10** 的 BOM 均消耗原材料 **Y00015**（共用料）。

**配置**

| 配置项 | 设置 |
|--------|------|
| 原材料 **Y00015** | 「是否冲减库存」= **是** |
| **Y00015** 对产业目录供给比例 | 目录 **A** : 目录 **B** = **70% : 30%** |
| 产品 **A1** 所用原材料（或 A1 展开至 Y00015 的路径） | 「是否冲减库存」= **否**（仅对 A1 相关展开需求不冲减） |

**假设需求与库存（便于算出 POC 描述的 700 / 300）**

| 输入 | 数值（示例） |
|------|----------------|
| 当期可用于冲减的 **Y00015** 总可用库存 | **1000** |
| E9 对 Y00015 的**毛需求**（BOM×批量后） | **500** |
| E10 对 Y00015 的**毛需求** | **500** |
| 两目录对 Y00015 的毛需求合计 | **1000**，与可用库存相等，可全额由库存满足 |

**推演过程（文字数据流）**

1. **MRP 读入**：客户/预测需求 → 展开 BOM → 得到各产品对 **Y00015** 的毛需求行（区分为 A1、E9、E10 等需求来源）。  
2. **A1 路径**：因 **Y00015**（或规则指定的层级）对 A1 **不冲减** → A1 侧仍保留按 BOM 算出的物料需求结果（净需求不因库存而扣减；POC 期望即「不进行库存冲减」的表现）。  
3. **E9 / E10 路径**：毛需求各 500，合计 1000；**Y00015** 对 A、B 目录配置比例 **70:30**。在「先满足毛需求、再按目录比例切分库存扣减额度」的一类算法下：可用于 E9+E10 的 1000 库存，按目录维度记账为 **A 侧消耗/占用 700**、**B 侧 300**（与 POC「A 剩余 700、B 剩余 300」的展示语义一致；若系统展示为「剩余可分配池」而非「剩余库存」，以实现字段为准）。  
4. **输出**：E9、E10 对 Y00015 **净需求为 0**（库存已覆盖）；报表或库存分析中 **Y00015** 在产业目录 **A**、**B** 下分别体现 **700** 与 **300** 的归属或结余结构。

**与不冲减分支的对比（同一推演中的另一条线）**

- 若将 **A1** 路径上的原材料也改为「冲减」，则 A1 展开出的 Y00015 需求会参与扣减可用库存，改变 E9/E10 可分得数量；POC 要求 **A1 不冲减**，故推演中 A1 不参与对 Y00015 可用池的竞争（或仅产生不计冲减的净需求展示）。

## 5. POC 假设与依赖（约束说明）

- **R1**：POC 假设物料—工厂—产线 **1:1:1**，可与当前「分到产线」模型对齐；若未来多产线，比例维度需扩展，本文档不展开。  
- **R2**：产业目录 ≡ 物品系列；产品与系列 POC 为 **1:1**；多产品可共用同一原材料。
