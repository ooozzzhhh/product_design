> 原型文件：`透视宽表报表配置.html`（直接用浏览器打开即可交互）
> 对应问题分析报告：`产品设计/产品问题分析/SDS/SDS_动态透视宽表框架设计/问题分析报告.md`

---

## 一、设计背景与核心思想

### 为什么要做这个框架？

SDS 模块有两类「时段宽表」报表：

| 报表 | 现状问题 |
|------|---------|
| 供应汇总（supplySummary） | 新增供应来源需改 6 处代码；SQL 有注入风险；维度硬编码 |
| 供需明细汇总（supplyAndDemandSummary） | 指标行和公式全部硬编码 |

两类报表有 60% 以上的公共逻辑（时间轴、分页、quantityMap 渲染），但代码完全独立。每当项目需要定制，就要改产品代码，导致**代码分叉、SDK 升级困难**。

### 核心设计思想

参考 **Excel 数据透视表**的五个区域拆解报表定义：

```
数据源（哪些表）
  ↓
行区域（GROUP BY 哪些维度 / 固定哪些指标行）
列区域（时间轴：按时段展开为列）
值区域（聚合什么字段 / 用什么公式）
筛选区域（业务用户的过滤条件）
```

五个区域全部配置化，由实施顾问在界面操作，业务用户只能查看和筛选。

## 二、两种引擎的区别

| 对比项 | 聚合引擎（AGGREGATION） | 指标树引擎（INDICATOR_TREE） |
|--------|------------------------|------------------------------|
| 行的来源 | 用户运行时动态选择维度，GROUP BY 后每组一行 | 固定指标树（期初/毛需求/净需求/期末…），行是预定义的 |
| 值的计算 | `SUM/COUNT/AVG` 等简单聚合 | 有跨行公式（净需求 = 毛需求 - 期初 - 非计划供应）和跨期滚动（期初 = 上期期末） |
| 数据来源 | 直接查供应/需求数据库表 | 调用 Service 层（DemandService + SupplyService） |
| 能否零代码新增 | ✅ 完全可配，实施顾问操作即可 | ⚠️ 指标标签/顺序可配；公式由 SDK 实现，需升级 jar |
| 典型报表 | 供应汇总、需求汇总、库存汇总 | 供需明细汇总、供需平衡表 |

## 四、各 Tab 功能与操作说明

### Tab 1：基本信息

- **模板编码**：创建后不可修改，全局唯一
- **模板名称**：可修改，显示在界面标题
- **所属模块**：SDS / DFP / MPS 等，用于权限过滤
- **引擎类型**：创建时确定，不可更改（如需切换引擎，需重新创建模板）
- **分页维度**（仅指标树引擎）：每页展示一组完整指标树的分页单元，如「物品+库存点」
- **是否启用**：禁用后业务用户无法查询该报表

---

### Tab 2a：数据段（仅聚合引擎）

每个数据段对应 UNION ALL 中的一个分支，配置内容说明：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 段编码 | 程序标识，大写+下划线，创建后不可改 | `PURCHASE_ORDER` |
| 段名称 | 展示用中文名，同时用作 supply_type 列的翻译值 | `采购订单` |
| 来源表名 | 数据库表名，保存时校验白名单 | `sds_ord_purchase_order` |
| 参与分类 | 该段参与哪些供应分类（计划/非计划） | `[计划供应]` |
| 分类过滤条件 | 各分类下的 WHERE 条件，不填表示全量参与 | `order_flag = 'NO'` |
| 虚拟列映射 | 将来源表的物理列映射到框架标准列；列名相同可不填 | `supply_date → order_date` |
| JOIN 配置 | 关联主数据表（物品、库存点、系列），框架自动拼入 SQL | `LEFT JOIN mds_pro_product_base AS pb ON s.item_id = pb.id` |

**操作流程：新增一个供应来源**

1. 点击「新增数据段」
2. 填写段编码（如 `CUSTOM_SUPPLY`）和段名称
3. 填写来源表名（如 `sds_supply_custom`），点击**「探测列」**
   - 系统自动读取 `information_schema`，展示该表所有列
   - 自动辅助填充字段映射
4. 勾选「参与分类」（计划/非计划）
5. 根据需要填写该分类下的过滤条件（如 `order_flag = 'NO'`）；不填则全量参与
6. 检查字段映射（虚拟列 supply_date 对应哪个物理列）
7. 点击**「保存数据段」**
8. 新段出现在列表中，启用开关默认打开

**注意**：删除数据段需二次确认；被禁用的段不参与 UNION ALL 查询，但配置保留。

---

### Tab 2b：指标定义（仅指标树引擎）

以树形结构展示所有指标行，说明如下：

| 列 | 说明 |
|----|------|
| 指标编码 | 程序标识，在 SDK 公式中被引用，不可在此改 |
| 指标名称 | **双击可编辑**，修改后保存生效 |
| 公式 Bean | SDK 中注册的 Spring Bean 名称，不可修改 |
| 排序 | 点击 ↑↓ 按钮调整显示顺序 |
| 启用 | 禁用后该指标行不在报表中展示 |

**什么可以配置：** 指标名称（中文改名）、显示顺序、是否启用

**什么不能配置：** 公式逻辑（如净需求的计算规则）、指标编码、新增公式类指标

**想新增指标时：** 联系产品团队在 SDK 中实现新的 `IndicatorFormula` Bean → 项目升级 jar → 在此界面添加指标行并填写新的 `formula_bean` 名称

---

### Tab 3：字段池

**维度字段（聚合引擎）**

- 定义业务用户在查询时可选的「分组维度」下拉列表
- **「默认选中」**：勾选后业务用户打开报表时自动使用该维度
- 若有枚举类（如物品类型），框架自动将 code 翻译为 label 展示
- 字段 code 须在至少一个数据段的字段映射中存在

**度量字段（聚合引擎）**

- 定义被聚合的列（如供应数量）和聚合函数（SUM/COUNT/AVG 等）
- 通常只配置一个度量（quantity），多度量时每列都会出现在结果中

**指标树引擎**：该 Tab 的维度/度量部分不适用，行由指标定义决定。

---

### Tab 4：时间轴

定义报表列头的时段来源，三种类型：

| 类型 | 适用场景 | 关键参数 |
|------|---------|---------|
| **MDS Feign 接口** | 已在 MDS 中配置时段组的场景（SDS 当前使用） | Feign Bean 名、接口方法、时段类型 |
| **自然月** | 不依赖 MDS、按自然月展开的场景 | 展望月数 |
| **数据库时段表** | 自定义时段结构 | 表名、起始/结束时间列名 |

`periodEnd 处理`：设置为「调整为当天 23:59:59」时，框架会在时段匹配前将时段结束时间调整到当天末，确保 supply_date 落入时段的边界判断一致（修复现有 periodEnd 边界问题）。

---

### Tab 5：筛选条件

定义业务用户在查询时可填写的过滤条件，配置项说明：

| 配置项 | 说明 |
|--------|------|
| 字段编码 | 对应请求参数名，创建后不可改 |
| 字段类型 | ENUM（下拉选择）/ TEXT / DATE / NUMBER |
| 枚举类 | 类型为 ENUM 时填写，框架自动渲染下拉选项 |
| 是否必填 | 必填字段在前端强制校验，接口层同步验证 |

**注意**：筛选字段的值由业务用户在运行时填写，框架始终以 `#{}` 参数化方式传入 SQL，不会拼入 SQL 字符串，安全可靠。

## 五、安全设计说明

本框架的动态 SQL 生成有三层安全保障：

| 安全层 | 机制 |
|--------|------|
| **表名白名单** | `table_name` 在配置保存时校验，框架执行前再次验证，非白名单表名直接拒绝 |
| **列名白名单** | `field_mappings` 中的列名只允许字母、数字、下划线，不允许特殊字符 |
| **用户输入参数化** | 业务用户填写的所有筛选值（filterValues）均通过 MyBatis `#{}` 传入，彻底消除 SQL 注入 |

框架生成的 SQL 中，唯一来自「用户输入」的部分是筛选值，且已参数化。GROUP BY 字段来自字段池白名单（实施顾问配置），不接受用户任意传入。

## 七、常见操作场景

### 场景 1：项目需要新增一类供应来源（如销售出库表）

1. 进入「供应汇总」模板 → 数据段 Tab
2. 点击「新增数据段」
3. 填写表名 `sds_sales_outbound`，点击「探测列」
4. 检查/调整字段映射（supply_date 对应哪个列）
5. 选择参与分类，填写过滤条件
6. 保存 → 立即生效
7. **全程无需改代码，无需重新部署**

### 场景 2：业务需要在供应汇总中新增「仓库」维度

1. 进入「供应汇总」模板 → 字段池 Tab
2. 点击「新增维度」，填写 `warehouse_code` / `仓库编码`
3. 确认各数据段的字段映射中有 `warehouse_code` 列（否则需先在数据段映射中补充）
4. 保存后，业务用户的维度下拉中即出现「仓库编码」选项

### 场景 3：供需明细汇总中某指标的中文名需要修改

1. 进入「供需明细汇总」模板 → 指标定义 Tab
2. 找到目标指标行，**双击指标名称**进入编辑状态
3. 修改名称，回车或点击其他区域确认
4. 点击顶部「保存配置」

### 场景 4：需要新增一个完全独立的宽表报表

1. 点击左侧底部「新建报表模板」
2. 填写模板编码、名称、模块，选择引擎类型
3. 创建后进入空白模板，依次配置：数据段（或指标定义）→ 字段池 → 时间轴 → 筛选条件
4. 保存后，前端通过 `/api/report/{templateCode}/page` 接口即可查询

| 现有代码 | 迁移后的对应位置 |
|---------|---------------|
| `SupplySummaryServiceImpl.getTableNames()` | `rpt_data_segment` 配置表（哪些段参与哪个分类） |

| `calSupplyAndDemand()` 各公式 | SDK 中的 `IndicatorFormula` Bean（不变，项目不分叉） |
