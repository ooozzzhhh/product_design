# 自动归档功能原型设计说明

## 1. 功能说明

- **功能定位**：提供数据库层面的「自动归档任务管理」原型，用于展示如何在前端配置与查看数据库归档任务。
- **核心能力**：
  - 管理归档任务列表：任务名称、归档对象（表）、触发条件、状态与操作；
  - 配置新建/编辑归档任务：基础信息、触发规则、归档对象与范围、归档后清理策略；
  - 查看归档任务执行日志：归档文件名、执行时间、状态、文件路径、失败原因等。
- **术语统一**：界面上所有中文用语已统一使用「归档」，不再使用「备份」一词；仅在文件名、路径或英文标识中保留 `backup` 作为历史命名（如 `backup_20241218_020000.sql`、`/data/backup/` 等），以避免影响现有技术方案和样例数据。

## 2. UI 样式

- 整体采用与运行监控中心一致的轻量化后台管理风格：
  - 顶部标题 + 工具条（场景切换、搜索输入框、导入/导出按钮、新建按钮）；
  - 中部为表格列表，使用浅灰表头、分隔线与标签样式展示状态；
  - 弹窗采用居中白底卡片，带阴影和圆角。
- 列表与弹窗控件统一使用：
  - 按钮：主要按钮为蓝色，次要按钮为白底灰边；
  - 表单：输入框和下拉框统一边框与圆角样式，获取焦点时高亮；
  - 标签：状态使用绿色/红色徽标；危险操作区域使用红色虚线边框与浅红背景。

## 3. UX 交互

### 3.1 主页面：数据库自动归档管理

- 顶部左侧展示标题「数据库自动归档管理」，右侧为：
  - 场景切换下拉（支持选择生产/测试/开发等数据库场景）；
  - 搜索框：按任务名称或表名关键字过滤列表；
  - 「导入」「导出」按钮：预留批量导入任务配置、导出模板与任务列表的交互位；
  - 「新建归档任务」按钮：打开配置弹窗。
- 列表列：
  - 任务名称；
  - 归档对象（表）：以逗号分隔的表名或简要概括；
  - 触发条件：展示「每周/每月/每季度/每年」+ 时间，或数据量触发条件摘要；
  - 状态：用标签标识是否启用；
  - 操作：运行、编辑、日志。
- 用户点击「运行」后暂以弹窗提示“归档任务已启动！”，实际进度展示由后端方案决定。

### 3.2 新建/编辑归档任务弹窗

弹窗划分为四个区域，按从上到下顺序展示：

1. **基础信息（Info）**
   - 任务名称：文本输入框，要求唯一且可读；
   - 存储格式：下拉选择（SQL 文件、NB3 等）；
   - 存储路径：文本输入框 +「校验权限」按钮，便于在配置阶段检查写权限；
   - 文件命名规则：单选项，例如 `{task_name}_{timestamp}.{ext}`、`{db_name}_{timestamp}.{ext}`。
2. **触发规则（When）**
   - 单选：定时触发 / 数据量触发；
   - 定时触发：
     - 时间颗粒度下拉：每周、每月、每季度、每年；
     - 不同颗粒度下动态展示对应的选择控件（如星期几、月初/月中/月末、月份+日期等）与时间选择；
     - 支持添加多条同一颗粒度下的触发条件。
   - 数据量触发：
     - 配置总行数阈值，超过阈值时触发归档。
3. **归档对象选择与范围配置（Who & What）**
   - 搜索框：按表名关键字搜索表，结果下拉供点击添加；
   - 已选列表：
     - 每行展示表名、删除按钮；
     - 行内配置：
       - 归档方式下拉：按时间 / 按数据量 / 按 SQL 条件；
       - 基准字段输入框（按时间/按数据量时展示）；
       - 不同归档方式对应的范围配置区：
         - 按时间：输入数字 + 单位（日/月/年） +「之前的数据」提示；
         - 按数据量：输入行数，描述为“按基准字段降序排序后，从第 N 行之后归档”；
         - 按 SQL 条件：条件行列表（字段 + 比较符 + 值）+「添加条件」按钮。
   - 批量设置基准字段：底部提供统一设置入口，一次性应用到所有已选表的基准字段。
4. **归档后操作（Delete）**
   - 勾选项「归档成功后清除已归档数据（危险操作）」；
   - 展开后：
     - 单选：自动清理 / 手动配置规则；
     - 手动模式下：规则行列表（表 + 时间字段 + 比较符 + 值）+「添加规则」按钮。

### 3.3 日志弹窗

- 在主页面点击「日志」按钮打开；
- 表头列：
  - 归档文件名；
  - 执行时间；
  - 状态（成功/失败）；
  - 文件路径；
  - 失败原因；
- 示例数据使用带 `backup_YYYYMMDD_HHMMSS.sql` 命名的文件，状态通过绿色/红色标签区分。

## 4. 实现注意事项

- **术语一致性**：实现时前后端、配置与文档中，应统一将此能力称为「自动归档」或「归档任务」，避免与传统数据库“备份/恢复”混淆。
- **数据安全与风险控制**：
  - 归档后清理属于高风险操作，正式实现时建议增加权限与审批控制；
  - 建议在后端层面对删除操作做分批执行与事务控制，避免对在线业务产生明显影响。
