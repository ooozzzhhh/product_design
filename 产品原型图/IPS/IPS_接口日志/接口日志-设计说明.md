# 接口日志 · 设计说明

---

## 一、功能说明

### 1.1 定位与背景

**功能定位**：为 IPS DCP 数据交换模块提供统一的「接口日志」入口，解决用户**无法看到每一条数据处理明细**、**页面与功能切割不合理**的问题。

**核心能力**：
1. **从外部获取数据**：向外部系统发起请求并接收数据的调用日志，支持自动重试与手动重试
2. **向外部提供数据**：外部系统向本系统发起请求，本系统返回数据的日志，展示**下发行数**（本系统下发的行数，不记录外部解析/应用情况）
3. **中间表**：仅记录从外部系统获取数据时回传的原始数据凭证，全量接收无拒收，不做任何处理
4. **场景库分发记录**：展示中间表数据分发到各场景库时的总数量/成功/失败统计，以及**因错误而丢失或拒绝录入的数据明细及具体错误原因**

### 1.2 与现有实现的对应关系

| 现有模块 | 本原型对应 | 说明 |
|---------|------------|------|
| extApiLog（接口日志） | 从外部获取数据 + 向外部提供数据 | 需区分调用方向，向外部提供数据的日志需后端扩展支持 |
| itf（场景接口数据） | 场景库分发记录 + 入库记录 | 需新增入库统计、分发统计及失败明细表 |

## 二、UI 样式

### 2.1 视觉方向

- **风格**：白色 + 天蓝色，简洁专业，适合数据监控与运维场景
- **主题色**：`#0ea5e9`（天蓝）作为主色，`#22c55e`（成功绿）、`#ef4444`（失败红）、`#f59e0b`（警示黄）作为状态色

### 2.2 色彩与字体

- **背景**：主背景 `#ffffff`，表头/分页区 `#f8fafc`，边框 `#e2e8f0`
- **字体**：Outfit（标题与正文）、JetBrains Mono（数字与代码块）、Noto Sans SC（中文）
- **状态**：成功绿、失败红、待处理灰、警示黄，配合小圆点与文字

### 2.3 组件规范

- **单页 Tab**：顶部标题 + Tab 切换（概览 | 从外部获取数据 | 向外部提供数据 | 中间表 | 场景库分发），合并为一个菜单入口
- **统计卡片**：4 列网格，可点击跳转对应 Tab
- **表格**：浅灰表头、悬停高亮、可点击行下钻
- **抽屉**：右侧滑出，560px 宽，用于详情与失败明细展示

## 三、UX 交互

### 3.1 信息架构

```
接口数据处理记录（单页 Tab 菜单）
├── 概览
│   ├── 4 个统计卡片（获取/提供/存储/分发，均展示次数：成功/失败 或 全部成功/部分成功/失败）
│   └── 最近调用记录表（混合类型，可点击下钻）
├── 从外部获取数据（向外部发起请求并接收数据）
│   ├── 筛选：接口名称、时间范围、状态
│   ├── 统计：总调用、成功、失败、平均响应（排除「—」不参与计算）
│   ├── 重试：失败记录支持自动重试（系统配置）与手动重试（按钮触发）
│   └── 列表 → 点击行/详情 → 抽屉展示请求头/体（响应体暂不存储）
├── 向外部提供数据（外部向本系统请求，本系统返回数据）
│   ├── 筛选：接口名称、时间范围、状态
│   ├── 统计：总调用、成功、失败、平均响应；下发行数（本系统下发的行数，不记录外部解析/应用）
│   └── 列表 → 点击行/详情 → 抽屉展示
├── 中间表（凭证记录，全量接收无拒收）
│   ├── 筛选：接口分类、接口日志ID（级联，均支持模糊搜索）
│   └── 大表格：展示该日志下所有中间表原始数据（数据ID、数据内容、入库时间）
└── 场景库分发记录
    ├── 筛选：接口分类、场景、接口日志ID、是否有失败
    ├── 汇总表：点击行联动下方失败明细表，失败行显示「手工重试」按钮
    ├── 分发明细表：展示当前选中汇总行的成功与失败数据（数据ID、状态、错误类型、错误详情）
    └── 手工重试：失败记录行显示「手工重试」按钮，点击后弹出转圈状态弹窗，2秒后随机成功/失败；成功时失败行数归零、失败原因摘要移除、分发时间更新
```

### 3.2 核心交互

| 场景 | 交互方式 | 说明 |
|------|----------|------|
| 切换视图 | 顶部 Tab 点击 | 概览 / 从外部获取数据 / 向外部提供数据 / 中间表 / 场景库分发 五类 Tab |
| 概览跳转 | 统计卡片点击 | 点击卡片直接切换到对应 Tab |
| 导出失败数据 | 导出按钮点击 | 导出当前筛选条件下所有失败数据（Excel） |
| 日志详情 | 表格行点击 / 详情链接 | 打开右侧抽屉，展示请求头/体、响应体；各支持复制按钮 |
| 中间表查询 | 选择接口分类 → 接口日志ID 下拉（支持模糊搜索）→ 查询 | 展示该日志下所有中间表原始数据 |
| 手工重试 | 场景库分发失败行「手工重试」按钮 | 弹出转圈状态弹窗，2秒后随机成功/失败；成功时失败行数归零、失败原因摘要移除、分发时间更新 |
| 汇总联动明细 | 场景库分发汇总表行点击 | 下方失败明细表联动展示该行的失败数据（数据ID、错误类型、错误详情） |
| 失败数据下钻 | 抽屉内错误列表 | 每条失败记录展示：数据ID、错误类型、具体错误信息 |

### 3.3 数据明细展示要求

**中间表**：仅展示原始数据凭证，无成功/失败区分，表格列：序号、数据ID、数据内容、入库时间。

**场景库分发失败明细**（每条失败记录需展示）：
- 数据ID
- 错误类型（外键约束、唯一约束、业务规则校验等）
- 错误详情（具体约束或规则说明）

## 四、实现时需要特别注意的细节

### 4.1 后端扩展

1. **入站调用日志**：当前 ext_api_log 主要记录出站，需扩展或新增表支持「外部调我们」的日志记录，并区分 `call_direction`（outbound/inbound）
2. **中间表入库统计**：需新增入库统计表或聚合逻辑，记录每次 api_log 对应的中间表写入：总数量、成功数、失败数，以及失败明细表（数据ID、错误类型、错误详情）
3. **场景库分发统计**：需新增分发统计表或聚合逻辑，记录每次从 itf 到场景库的写入：总数量、成功数、失败数，以及失败明细表
4. **失败明细存储**：建议在写入/分发时同步写入失败明细，避免事后无法追溯

### 4.2 与现有页面整合

- 现有「接口日志」页面（`/dc/system/apiLog`）可合并为本原型的「出站调用日志」视图
- 现有「场景接口数据」页面（`/dc/system/sceneInterfaceData`）可作为本原型的补充：从入库/分发明细中的「跳转场景接口数据」可带 apiLogId 参数跳转

### 4.3 性能与分页

- 失败明细可能较多，建议分页或懒加载
- 概览「最近调用记录」可限制为最近 N 条（如 20 条），按时间倒序

### 4.4 导出与告警

- 建议支持导出失败明细为 Excel，便于运维排查
- 失败数量超过阈值时可考虑与告警联动（非本原型范围，可后续扩展）

### 4.5 错误明细大量数据时的处理

- **分页**：失败条数超过 100 时，仅展示前 50 条，支持分页浏览
- **导出**：失败条数超过 1000 时，建议引导用户导出 Excel 后离线分析
- **展示提示**：在错误明细区域顶部显示「共 N 条失败，当前展示前 M 条」及导出按钮

### 4.6 失败原因摘要枚举（供开发参考）

以下为可预见的、较常见的失败原因，便于开发人员在实现失败明细时统一归类与展示：

**一、网络与连接类**
| 失败原因 | 说明 | 典型场景 |
|----------|------|----------|
| 连接超时 | 建立连接或等待响应超时 | 外部系统不可达、网络抖动 |
| 连接拒绝 | 目标端口未监听或拒绝连接 | 外部服务未启动、防火墙拦截 |
| DNS 解析失败 | 无法解析目标域名 | 域名配置错误、DNS 故障 |
| SSL/TLS 握手失败 | 证书校验或协议版本不匹配 | 证书过期、双向认证配置错误 |

**二、HTTP 与协议类**
| 失败原因 | 说明 | 典型场景 |
|----------|------|----------|
| HTTP 4xx 客户端错误 | 请求参数或鉴权问题 | 400 参数错误、401 未授权、403 禁止、404 不存在 |
| HTTP 5xx 服务端错误 | 外部系统内部异常 | 500 内部错误、502 网关错误、503 服务不可用 |
| 响应格式异常 | 返回非预期格式（非 JSON、结构不符） | 外部接口变更、错误页返回 HTML |
| 响应体为空 | 成功状态码但无有效数据 | 外部系统逻辑异常 |

**三、数据校验与业务类（中间表入库 / 场景库分发）**
| 失败原因 | 说明 | 典型场景 |
|----------|------|----------|
| 主键重复 | 插入数据主键已存在 | 重复推送、幂等未处理 |
| 唯一约束冲突 | 唯一索引字段重复 | 业务编码重复、联合唯一约束 |
| 外键约束 | 关联的主表记录不存在 | 库存点 ID 不存在、物料主数据未同步 |
| 字段校验失败 | 必填、格式、长度等校验不通过 | 物料编码为空、日期格式错误 |
| 数据类型不匹配 | 字段类型与表定义不符 | 数字传字符串、日期格式错误 |
| 字段长度超限 | 超出数据库字段长度 | 长文本截断、VARCHAR 超长 |
| 业务规则校验失败 | 自定义业务规则不满足 | 状态流转非法、数量为负 |

**四、解析与转换类**
| 失败原因 | 说明 | 典型场景 |
|----------|------|----------|
| JSON 解析失败 | 响应体非合法 JSON | 截断、编码错误、混入非法字符 |
| 字段映射缺失 | 必填字段在源数据中缺失 | 接口契约变更、上游未传 |
| 数据转换异常 | 类型转换、枚举映射失败 | 枚举值不在字典内、数字格式错误 |

**五、系统与资源类**
| 失败原因 | 说明 | 典型场景 |
|----------|------|----------|
| 数据库连接池耗尽 | 无法获取数据库连接 | 高并发、连接泄漏 |
| 磁盘空间不足 | 写入失败 | 日志或临时文件占满 |
| 内存不足 | OOM 或分配失败 | 大批量数据处理 |
| 线程池/队列满 | 异步任务无法提交 | 下游处理能力不足 |

**实现建议**：失败原因摘要字段建议存储上述枚举值或可扩展的编码，便于统计、筛选与告警规则配置。
