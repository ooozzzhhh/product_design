# 运行监控中心 - 产品需求说明书（PRD）

> **版本**: v2.0  
> **日期**: 2026-02-09  
> **适用范围**: 悠桦林 SCP 平台各业务模块（SCH、MPS、S&OP、DFP、MRP、MDS、SDS、IPS 等）

---

## 1. 需求概述

### 1.1 背景及问题（为什么要做）

- 当前各业务模块（MRP/SCH/MPS/S&OP/DFP 等）中，一次性长时间复杂计算（如历史需求清洗、供需匹配、排程计算、报告生成）分散在不同菜单和页面，**运行入口分散、监控方式不统一**。  
- 运行结果的查看方式不一致：有的只有简单日志，有的只有任务列表，**缺乏统一的任务视图和状态语义**（成功/运行中/失败/已终止等）。  
- 对运维和业务人员而言，跨模块排查问题、追踪任务链路成本高，**无法一眼看清「今天跑了什么」「哪些还在跑」「哪些失败/被终止」**。  
- 现有前端在 MRP 领域已有一套较成熟的「运行结果中心」设计，但**未平台化、无法被其他模块复用**。

### 1.2 目标（期望效果）

- 建立平台级「运行监控中心」，将各模块的一次性长时任务统一到**同一交互范式和视觉规范**下。  
- 通过**功能卡片 + 功能详情页 + 运行进度弹窗 + 执行历史列表**的标准结构，让业务人员：  
  - 快速定位某个功能（如「安全库存计算」「规则式排程」「产销计划报告」）；  
  - 看到该功能最近的运行情况（今日/本周次数、成功率、平均耗时）；  
  - 方便地发起运行、查看进度、终止任务、下载算法文件或报错日志。  
- 支持**高数据量场景**（大量历史记录）下的分页和筛选，保证体验和性能。  
- 为后续与后台服务整合预留统一接口规范，便于逐步从「前端模拟」演进为「真实生产」监控中心。

---

## 2. 详细解决方案

### 2.1 描述（粗描述）

- 在 SCP Web 前端构建一个统一入口的「运行监控中心」页面：  
  - 顶部为模块切换（SCH/MPS/S&OP/DFP/MRP/IPS 等）；  
  - 中间展示当前模块下的**功能卡片网格**；  
  - 卡片上展示功能名称、描述、最近一次运行状态与时间、今日/本周次数、使用用户数等；  
  - 点击卡片进入**功能详情页**，查看运行概览、执行历史、输入信息、算法日志、计算结果、AI 总结等。  
- 在功能详情页中，用户可：  
  - 打开**运行配置弹窗**，配置策略名称、运行模式（增量/全量）、策略模式（规则/优化）、差异化业务参数；  
  - 发起运行后，自动弹出**运行进度弹窗**，以「步骤时间线 + 进度条 + 当前节点转圈」方式展示执行情况，并允许人工终止任务；  
  - 在执行历史列表中使用**搜索、日期、状态筛选、分页**等能力快速定位任务，并展开单条任务查看详细输入、日志和结果。  
- 所有模块通过统一的 `MODULE_CONFIGS / FUNCTION_CONFIGS / RUN_ACTION_CONFIGS` 配置接入运行监控中心，做到**「加配置即可接入」**。

### 2.2 输入（可不填）

> 说明：本节描述从业务和系统角度，该能力依赖的「输入信息」，后端对接时可据此完善接口。

- **业务输入**（用户在运行配置弹窗中选择或录入）：  
  - 需求预测模块（DFP）：数据源、数据范围、清洗规则、需求版本、预测维度、模型类型等；  
  - 产销协同模块（S&OP）：计划版本、报告周期、会议 ID/主题等；  
  - 主计划/排程模块（MPS/SCH）：排程规则、订单范围、产线/工序、负载阈值等；  
  - MRP：BOM 层级、仓库、服务水平目标、安全库存策略等；  
  - IPS：同步方向、目标系统、归档范围与保留期限等。  
- **系统输入**（后端侧）：  
  - 历史需求数据、计划版本与需求版本主数据；  
  - 订单、BOM、产能、库存、接口配置等业务数据；  
  - 用户权限、角色信息（控制运行/终止/下载权限）。

### 2.3 输出（可不填）

- **前端呈现的主要输出**：  
  - 任务执行结果摘要（成功/失败/已终止 + 运行耗时 + AI 总结）；  
  - 算法级别的关键指标（如：清洗后条数、预测 MAPE、齐套率、安全库存结果等）；  
  - 输入信息快照（本次运行所用的版本号、时间范围、规则参数等）；  
  - 算法运行日志（按时间线展示关键步骤）；  
  - 执行节点时间线（每个步骤的完成时间与默认备注）。  
- **对系统的输出**（后续接入后端时）：  
  - 持久化运行记录（RunRecord），供运维、审计和报表使用；  
  - 中台/算法服务可基于统一 RunRecord ID 反查详细结果数据。

### 2.4 业务流程

#### 2.4.1 模块切换与功能发现

1. 用户通过顶部模块下拉选择业务模块（如 MRP、SCH、S&OP）。  
2. 页面加载该模块的功能卡片列表，仅展示当前模块的功能。  
3. 卡片统计区显示：今日运行次数、本周运行次数、成功率、平均耗时等。  

#### 2.4.2 发起运行

1. 用户在功能详情页点击「运行 XXX」按钮（按钮文案/图标由 `RUN_ACTION_CONFIGS` 差异化配置）。  
2. 弹出「运行配置」弹窗：  
   - 展示通用配置：运行模式（增量/全量）、策略模式（规则/优化）、策略名称等；  
   - 展示功能差异化必选参数（下拉/文本/日期等）。  
3. 用户点击「确认并开始」，前端生成一条 `status=running` 的 RunRecord，写入执行历史列表。  
4. 自动关闭配置弹窗，并打开运行进度弹窗。  

#### 2.4.3 查看进度（运行中）

1. 运行进度弹窗按照**步骤时间线**展示当前执行节点和整体进度（以「当前步骤/总步骤」形式展示，不再使用百分比）。  
2. 运行中任务在功能详情页历史列表中标记为「运行中」，进度以步骤数分数展示（如 2/4）。  
3. 用户可以：  
   - 直接关闭弹窗，仅隐藏进度视图，任务继续运行；  
   - 点击右上角「终止」按钮，任务状态变为「已终止」，在列表中以黄色主题和减号图标展示。  

#### 2.4.4 查看结果（成功）

1. 历史列表中成功任务使用绿色主题和勾号图标展示。  
2. 用户点击任务行右侧的「算法文件」按钮，可下载本次运行的配置/策略快照。  
3. 展开任务详情，可查看输入信息、算法运行日志、计算结果及 AI 总结。  

#### 2.4.5 排查失败/异常任务

1. 失败任务使用红色主题和 X 图标展示。  
2. 用户可点击「报错日志」按钮下载 CSV 错误明细（含错误级别、模块、错误代码、描述等）。  
3. 展开详情可查看算法日志、输入信息和部分输出，辅助排查根因。  

#### 2.4.6 执行节点查看

1. 用户在历史列表中点击「节点」按钮，打开执行节点详情弹窗。  
2. 弹窗内按顺序展示各执行步骤的状态：未执行/执行中/已完成/失败/已终止。  
3. 为运行中和终止场景，突出当前节点，并显示默认备注「执行中… / 已终止」。  

#### 2.4.7 筛选与分页

1. 用户可按任务 ID、操作人关键字搜索。  
2. 支持按开始日期范围筛选（起止日期），默认展示最近一段时间。  
3. 状态筛选包括：全部 / 成功 / 失败 / 运行中 / 已终止。  
4. 支持分页：可选择每页 10/20/50/100 条，并通过上一页/下一页切换页码，底部展示「当前范围 + 总条数」。  

### 2.5 业务规则

- **模块与功能接入规则**  
  - 所有接入模块必须在 `MODULE_CONFIGS` 中配置模块 ID、名称、运行 ID 前缀、描述及图标；  
  - 每个功能在 `FUNCTION_CONFIGS` 中声明：功能 ID、所属模块、名称、描述、是否算法功能、统一图标等；  
  - 运行按钮行为（文案、图标、必选参数、是否展示策略配置）由 `RUN_ACTION_CONFIGS` 统一管理。  

- **任务 ID 与版本规则**  
  - 任务 ID 前缀建议与模块保持一致：SCH/MPS/SOP/DFP/MRP/MDS/SDS/IPS；  
  - 有计划/需求版本概念的模块（MPS/S&OP/DFP/MRP 等），在 RunRecord 中必须记录 `planVersion` / `demandVersion`，并在卡片和历史列表中展示。  

- **状态机规则**  
  - 状态枚举：`running` / `success` / `failed` / `terminated`；  
  - 正常运行结束：`running → success`；  
  - 系统异常：`running → failed`；  
  - 人工终止：`running → terminated`，任务不再继续执行，列表中显示黄色主题与「已终止」标记。  

- **权限与安全规则**  
  - 运行、终止、下载算法文件、下载报错日志等操作需校验用户权限；  
  - 报错日志可能包含敏感信息（如业务数据主键、内部错误堆栈），导出前需做脱敏与权限控制。  

- **性能与数据量规则**  
  - 历史列表必须采用分页加载，默认每页 20 条，可根据业务选择 10/20/50/100；  
  - 当历史数据量巨大时，建议后端提供分页游标或服务端排序，前端可进一步引入虚拟滚动。  

### 2.6 原型图（描述交互逻辑）

> 说明：本节用文字描述关键界面和交互，具体 HTML 原型见 `产品前端UI设计/运行监控中心` 目录。

- **首页（模块功能卡片列表）**  
  - 顶部：左侧为运行监控中心标题与图标，右侧为模块下拉选择；  
  - 中部：按网格展示当前模块下的功能卡片，每张卡片包含：  
    - 左侧大图标（按模块/功能差异化）；  
    - 右侧两行文案：第一行功能名称（若有运行中任务则右侧显示加载圈），第二行为简要描述；  
    - 下方为统计信息（最近运行时间、今日/本周次数、本周使用人数、平均耗时）。  

- **功能详情页**  
  - 顶部：返回功能列表按钮 + 功能名称；右侧为运行按钮（按功能配置文案和图标）；  
  - 中部：运行概览卡片，展示今日/本周运行次数、成功率、平均耗时等；  
  - 下方：筛选区域（搜索、日期、状态、重置）+ 执行历史列表（可展开查看输入/日志/结果），底部分页条。  

- **运行配置弹窗**  
  - 标题采用 `RUN_ACTION_CONFIGS[functionId].modalTitle`；  
  - 分为通用区域（运行模式/策略模式/策略名称）与差异化参数区域（按功能配置）；  
  - 统一使用 SCP 当前 Tailwind 风格与输入控件。  

- **运行进度弹窗**  
  - 顶部左侧为标题（「正在执行计算」/「查看运行进度」），右侧在查看运行中任务时展示「终止」按钮；  
  - 中部为整体进度条，以步骤分数形式展示当前进度（如 2/4），并配合涌动动画；  
  - 下部为步骤时间线，每个步骤有圆点状态与说明文案，当前步骤使用旋转图标和「计算中…」文案突出。  

### 2.7 测试用例

> 这里只列关键业务回归用例，详细字段级用例可由测试同学进一步拆分。

- **用例 1：模块切换与功能卡片展示**  
  - 前置：存在多个模块的功能配置与运行历史模拟数据；  
  - 步骤：依次切换 SCH/MPS/S&OP/DFP/MRP 等模块；  
  - 预期：每个模块的功能卡片展示正确，卡片统计信息与该模块历史记录一致。  

- **用例 2：发起运行并查看进度**  
  - 步骤：在某功能详情页点击「运行 XXX」，在配置弹窗填写必要参数并确认；  
  - 预期：产生一条 `status=running` 的新记录，自动弹出进度弹窗，进度条与步骤节点随时间推进，执行完成后状态变为 `success`，历史列表与概览刷新。  

- **用例 3：运行中终止任务**  
  - 步骤：在运行进度弹窗点击右上角「终止」；  
  - 预期：进度弹窗关闭，对应 RunRecord 状态变为 `terminated`，历史列表使用黄色主题与减号图标展示，时长列显示「已终止」，筛选中的「已终止」能正确过滤该记录。  

- **用例 4：失败任务报错日志下载**  
  - 步骤：在执行历史中找到失败任务，点击「报错日志」；  
  - 预期：浏览器下载 CSV 文件，内容包含错误级别、模块、错误代码、异常描述等，文件编码正确（支持中文）。  

- **用例 5：筛选与分页**  
  - 步骤：  
    1. 通过任务 ID/操作人关键字搜索；  
    2. 限定日期范围；  
    3. 切换不同状态过滤（成功/失败/运行中/已终止）；  
    4. 调整每页条数为 10/50/100，并用上一页/下一页切换页面；  
  - 预期：列表展示的数据始终与筛选条件、分页参数一致，底部统计「显示范围 + 总条数」正确。  

- **用例 6：节点详情展示**  
  - 步骤：在成功、失败、已终止、运行中等不同状态的任务上点击「节点」按钮；  
  - 预期：弹窗中各步骤状态正确，高亮当前/失败/终止节点，备注文案符合业务含义。
