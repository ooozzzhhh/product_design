# 1. 需求概述

## 1.1 背景与问题

当前系统有「接口管理」功能，包含接口配置、接口同步控制、接口日志、接口数据四个模块。数据流程是：**外部系统 → 中间表 → 场景库**。

**现有问题**：
- 无法清晰看到「获取到中间表」和「分发到场景库」两个阶段的成功/失败情况
- 接口日志和接口数据分散在两个页面，查问题需要来回切换
- 失败时看不到具体是哪些数据出错、什么原因
- 场景库分发失败后无法手工重试

## 1.2 改造目标

将原「接口日志」和「接口数据」合并为新的「接口日志」页面，让用户能：
- 一个页面看全流程：从外部调用 → 中间表 → 场景库的完整链路
- 快速定位问题：哪个阶段失败、哪些数据失败、具体错误原因
- 导出失败数据，便于排查和修复
- 对场景库分发失败的记录进行手工重试

## 1.3 改造范围

**本次改造**：
- 新「接口日志」页面（5 个 Tab 视图）
- 失败数据明细展示与导出
- 场景库分发失败记录手工重试功能
- 从接口配置跳转到接口日志

**不改造**：
- 接口配置、接口同步控制页面保持现状
- 后端 ETL/引擎层逻辑（仅需补充日志采集能力）

# 2. 页面结构与交互

## 2.1 整体布局

新「接口日志」采用**单页 Tab 切换**设计，包含 5 个视图：

```
接口日志（单页）
├── 概览
├── 从外部获取数据
├── 向外部提供数据
├── 中间表
└── 场景库分发
```

## 2.2 概览 Tab

**用途**：快速了解整体情况，提供跳转入口。

**页面元素**：
1. **4 个统计卡片**（可点击跳转对应 Tab）
   - 从外部获取数据：显示调用次数（成功/失败）
   - 向外部提供数据：显示调用次数（成功/失败）
   - 中间表：显示入库次数（全部成功/部分成功/失败）
   - 场景库分发：显示分发次数（全部成功/部分成功/失败）

2. **最近调用记录表**（最近 20 条，按时间倒序）
   - 列：序号、接口名称、类型（获取/提供/分发）、状态（成功/失败/全部成功/部分成功/失败）、成功（行数）、失败（行数）、时间
   - 点击行可跳转到对应详情

**交互**：
- 点击统计卡片 → 切换到对应 Tab
- 点击表格行 → 打开详情抽屉

**状态展示规则**：
- 获取/提供类型：成功/失败（二态）
- 存储/分发类型：全部成功/部分成功/失败（三态）

## 2.3 从外部获取数据 Tab

**用途**：查看本系统向外部系统发起请求、获取数据的调用日志。

**筛选条件**：
- 接口名称（下拉选择）
- 时间范围（日期选择器，默认最近 7 天）
- 状态（全部/成功/失败）

**列表字段**：
- 序号
- 接口名称
- 批次号
- 请求方式（GET/POST）
- 响应状态（成功/失败，带颜色标识）
- 解析/应用（显示「解析数 / 应用数」，如「100 / 95」）
- 响应时间（毫秒）
- 错误信息（失败时显示）
- 重试（失败时显示「手动重试」按钮）
- 创建时间
- 操作（「详情」链接）

**交互**：
- 点击「详情」→ 打开右侧抽屉，展示：
  - 请求头（可复制）
  - 请求体（可复制）
  - 响应体（可复制）
  - 解析数量、应用数量
  - 错误信息（如有）
- 点击「手动重试」→ 重新发起该接口调用

## 2.4 向外部提供数据 Tab

**用途**：查看外部系统向本系统发起请求、本系统返回数据的日志。

**筛选条件**：
- 接口名称（下拉选择）
- 时间范围（日期选择器）
- 状态（全部/成功/失败）

**列表字段**：
- 序号
- 接口名称
- 来源系统
- 批次号
- 响应状态（成功/失败）
- 下发行数（本系统下发的数据行数）
- 响应时间（毫秒）
- 错误信息（失败时显示）
- 创建时间
- 操作（「详情」链接）

**交互**：
- 点击「详情」→ 打开右侧抽屉，展示请求/响应信息

## 2.5 中间表 Tab

**用途**：查看从外部获取的原始数据凭证（全量接收，无拒收）。

**筛选条件**：
- 接口分类（下拉选择）
- 接口日志 ID（级联下拉，支持模糊搜索）

**列表字段**：
- 序号
- 数据 ID
- 数据内容（JSON 格式）
- 入库时间

**说明**：
- 中间表仅作凭证记录，不区分成功/失败
- 选择接口分类后，接口日志 ID 下拉框自动加载该分类下的日志

## 2.6 场景库分发 Tab

**用途**：查看中间表数据分发到各场景库的情况，重点关注失败数据，支持手工重试。

**页面布局**（v1.7～v1.9 优化）：
- 筛选栏位于 Tab 说明右侧，与标题描述同一行
- 下方为汇总表（上）+ 分发明细表（下）的联动布局
- 移除 KPI 卡片行，简化页面层级

**筛选条件**：
- 接口分类（下拉选择）
- 场景（下拉选择，如库存、需求、订单等）
- 接口日志 ID（输入框，支持模糊搜索）
- 是否有失败（全部/仅失败）

**汇总表字段**：
- 序号
- 接口日志 ID
- 接口分类
- 场景
- 总数量
- 成功数
- 失败数
- 失败原因摘要（如「主键重复」「外键约束」等，失败时显示；成功时为「—」）
- 分发时间
- 操作（失败时显示「手工重试」按钮）

**分发明细表**（v1.8 新增）：
- 联动展示：点击汇总表某行时，明细表自动展示该行的所有分发数据（成功+失败）
- 列字段：
  - 序号
  - 数据 ID
  - 状态（成功/失败，带颜色标识）
  - 错误类型（失败时显示，如「主键重复」「外键约束」「字段校验失败」等；成功时为「—」）
  - 错误详情（失败时显示具体错误信息；成功时为「—」）
- 支持分页（超过 50 条时）

**交互**：
- 点击汇总表行 → 下方明细表联动展示该行的成功与失败数据
- 点击「手工重试」按钮（v1.6 新增）→ 弹出转圈状态弹窗，2 秒后随机成功/失败；成功时：
  - 失败行数归零
  - 失败原因摘要移除（显示为「—」）
  - 分发时间更新为当前时间
  - 明细表中原失败记录状态更新为成功，错误类型/详情清空
- 点击「导出失败数据」→ 导出当前筛选条件下所有失败数据（Excel）
- 全部成功时，失败数显示绿色「全部成功」，无导出按钮

**手工重试说明**：
- 仅失败记录显示「手工重试」按钮
- 点击后触发后端重新执行该批次的场景库分发逻辑
- 重试过程中显示转圈状态弹窗，禁止重复点击
- 重试结果：
  - 成功：汇总表失败数归零，失败原因摘要移除，分发时间更新；明细表中失败记录状态更新为成功
  - 失败：保持失败状态，错误信息更新为最新错误原因

# 3. 核心业务规则

## 3.1 数据流程阶段

数据流程分为 3 个阶段：

| 阶段 | 名称 | 说明 | 常见错误 |
|------|------|------|----------|
| A | 外部系统调用 | 本系统向外部发起请求或外部向本系统发起请求 | 连接超时、HTTP 4xx/5xx、参数错误 |
| B | 写入中间表 | 将接口响应数据解析并写入中间表 | 解析失败、主键重复、字段校验失败 |
| C | 分发到场景库 | 从中间表分发数据到各场景库 | 外键约束、唯一约束、业务规则校验失败 |

## 3.2 状态展示规则

**获取/提供类型（二态）**：
- **成功**：绿色标识，错误信息为空或「—」
- **失败**：红色标识，显示错误信息

**存储/分发类型（三态）**：
- **全部成功**：绿色标识，失败数为 0
- **部分成功**：黄色标识，成功数 > 0 且失败数 > 0
- **失败**：红色标识，成功数为 0

## 3.3 查询与过滤规则

- 默认查询时间范围：最近 7 天
- 最大查询历史区间：90 天（超出提示「请缩小时间范围」）
- 失败明细超过 50 条时分页展示，超过 1000 条时引导用户导出 Excel

## 3.4 日志保留策略

- 默认保留周期：30 天（可配置）
- 超期数据归档至历史表或冷存储

## 3.5 手工重试规则

- 仅场景库分发失败记录支持手工重试
- 重试触发后，后端重新执行该批次的场景库分发逻辑
- 重试成功后，汇总表与明细表数据同步更新
- 重试过程中禁止重复点击，显示加载状态

# 7. 后端支持需求

## 7.1 需要补充的能力

1. **入站调用日志**：当前主要记录出站（本系统调外部），需扩展支持「外部调本系统」的日志记录
2. **中间表入库统计**：记录每次接口调用对应的中间表写入：总数量、成功数、失败数、失败明细
3. **场景库分发统计**：记录每次从中间表到场景库的写入：总数量、成功数、失败数、失败明细
4. **失败明细存储**：建议在写入/分发时同步写入失败明细，避免事后无法追溯
5. **手工重试接口**：提供场景库分发失败记录的手工重试接口，支持按接口日志 ID + 场景重新执行分发逻辑

## 7.2 错误码与 traceId

若平台已实施「IPS_平台报错规范化优化」，接口日志中的错误信息应遵循统一规范：
- 错误码 + 用户可见文案 + traceId
- 详情抽屉中预留错误码、traceId 展示位置
