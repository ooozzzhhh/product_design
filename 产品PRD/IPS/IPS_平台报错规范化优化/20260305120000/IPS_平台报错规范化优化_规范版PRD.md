# 2. 规范转 PRD：异常处理技术规范

## 2.1 异常分类与业务大类编码

| 业务大类 | 编码前缀 | 说明 |
|----------|----------|------|
| 需求计划 | DP | Demand Planning |
| 产销协同计划 | SOP | Sales & Operations Planning |
| 主生产计划 | MPS | Master Production Schedule |
| 物料需求计划 | MRP | Material Requirements Planning |
| 主需求计划 | MDS | Master Demand Schedule |
| 生产排程 | SCH | Scheduling |
| 系统通用 | SYS | System |

## 2.2 异常枚举设计规范

- **枚举结构**：`BusinessErrorEnum`，包含 `code`（异常编码，关联 i18n key）、`desc`（默认描述）
- **编码格式**：`业务大类前缀 + 3位序号`，如 DP001、SOP001、MPS001、MRP001、MDS001、SCH001、SYS001
- **国际化**：通过 `I18nUtil.getString(code)` 加载文案，支持 `MessageFormat.format` 占位符
- **配置文件**：`messages_zh_CN.properties`、`messages_en_US.properties`

**错误码与文案维护位置**：

| 维护项 | 维护位置 |
|--------|----------|
| 错误码定义 | `BusinessErrorEnum` 枚举，`code` 字段（关联国际化配置 key） |
| 错误码与文案映射 | `messages_zh_CN.properties`（中文）、`messages_en_US.properties`（英文），key = 错误码，value = 文案模板（支持 `{0}`、`{1}` 占位符） |
| 加载方式 | 通过 `I18nUtil.getString(code)` 加载，`MessageFormat.format` 填充占位符 |

新增错误码时，须同时在枚举中定义、并在上述 properties 文件中补充对应 key-value。

## 2.3 异常抛出规范

| 规则 | 要求 |
|------|------|
| 抛出层级 | 业务异常**仅允许在 Service 层抛出**，禁止在 Controller、DAO 层抛出 |
| DAO 异常 | 需在 Service 层捕获，转换为业务异常或系统异常 |
| 无占位符 | `throw new BusinessException(BusinessErrorEnum.XXX.getI18nErrordesc())` |
| 带占位符 | `throw new BusinessException(BusinessErrorEnum.XXX.getI18nErrordesc(arg1, arg2))` |

**必须使用 `getI18nErrordesc()`**，禁止使用 `getDesc()`。

## 2.4 禁止行为

- 禁止直接抛出 `RuntimeException` / `Exception` 替代业务异常
- 禁止硬编码异常描述（必须通过枚举 + 国际化配置）
- 禁止在循环 / 高频调用中频繁抛出异常
- 禁止占位符参数数量与国际化模板不匹配
- 禁止将 `e.getMessage()`、`e.toString()`、堆栈、SQL 返回前端

# 3. 报错提示语句编写规范（规范中未提及，本 PRD 补充）

## 3.1 总体原则

- **用户可理解**：使用业务语言，避免技术术语（如「表结构异常」「voClassName」）
- **可操作性**：说明问题 + 指明对象 + 建议下一步（检查配置/联系管理员/稍后重试）
- **一致性**：术语统一（如统一使用「物料」而非「产品」「商品」）

## 3.2 按场景的文案模板

| 场景 | 模板 | 示例 |
|------|------|------|
| 参数缺失 | 「{参数名}」不能为空 | 「用户名」和「密码」不能为空 |
| 参数格式/范围 | 「{参数名}」格式不正确或超出有效范围，请检查后重试 | — |
| 配置缺失 | 尚未配置「{配置项}」，请前往「{菜单路径}」完成配置后重试 | 尚未配置「计划期间」，请前往主数据完成配置后重试 |
| 数据不存在 | {业务对象}不存在，请确认后重试 | 物料不存在，请确认后重试 |
| 数据已存在 | 「{对象标识}」已存在，请勿重复添加 | — |
| 状态不允许 | {业务对象}已{状态}，无法{操作} | 该工序已生产，不得调整 |
| 删除限制 | {业务对象}存在关联数据，无法删除，请先解除关联 | — |
| 并发冲突 | {业务对象}已被他人修改，请刷新后重试 | — |
| 权限不足 | 无权限执行该操作，请联系管理员 | — |
| 账号状态 | 账号已{停用/锁定}，请联系管理员 | — |
| 密码相关 | 密码{过期/不符合要求}，请先修改密码 | — |
| 导入/导出失败 | {操作}失败，请下载错误明细并按提示修正后重试 | 导入数据存在错误，请下载错误明细并按提示修正后重试 |
| 模板/格式不支持 | 当前{模板/文件格式}不支持在本模块使用，请使用正确模板后重试 | — |
| 任务运行中 | 计划/任务正在运行中，请稍后再试 | 计划正在运行中，请稍后再试 |
| 计算/排程失败 | {计划/排程}计算失败，请检查配置与数据后重试 | — |
| 流程异常 | 流程{启动/执行}失败，请稍后重试或联系管理员 | — |
| 外部服务失败 | 调用{服务名}失败，请稍后重试 | — |
| 请求超时 | 请求超时，请稍后重试 | — |
| 系统错误 | 系统繁忙，请稍后重试。如有问题请联系管理员，并提供错误码与追踪ID。 | — |

## 3.3 禁用表述

| 禁止 | 说明 |
|------|------|
| `e.getMessage()`、`e.toString()` 透传 | 技术信息仅写日志，不返回前端 |
| 「操作失败:」+ 异常信息 | 使用场景化固定文案 |
| 空泛「运行失败」「操作失败」无下一步 | 必须给出「稍后重试/联系管理员」等指引 |
| 参数缺失只说「不能为空」不指名称 | 使用「「{参数名}」不能为空」 |
| 配置缺失只说「未配置」不指路径 | 补充配置项名称或菜单路径 |
| 状态不允许只说「不允许」 | 补充当前状态与替代路径 |

## 3.4 术语统一

| 规范用词 | 禁用/混用 |
|----------|------------|
| 物料 | 产品、商品、物品（业务上下文明确指 SKU 时可保留） |
| 任务 | 作业、Job（日志内部可保留） |
| 参数 | 字段、变量 |

## 3.5 标点与格式

- 句尾统一使用中文标点（。！）
- 参数名、配置项名使用「」包裹
- 避免中英文标点混用（如 "sql语法错误," 应改为「SQL 语法错误」）

# 4. 用例（规范符合性验收）

| 用例 | 前置条件 | 操作步骤 | 预期结果 |
|------|----------|----------|----------|
| 抛出层级 | 业务校验失败 | 检查调用栈 | 异常在 Service 层抛出，Controller 仅调用 Service |
| 枚举+国际化 | 新增报错点 | 检查代码 | 使用 BusinessErrorEnum，文案在 messages_zh_CN.properties |
| 错误码维护 | 新增错误码 | 检查枚举与 properties | 枚举中已定义，properties 中已补充 key-value |
| 无技术透传 | 捕获异常后返回 | 检查响应 msg | 不包含 e.getMessage()、堆栈、SQL |
| 占位符匹配 | 带占位符异常 | 检查参数数量 | 与 i18n 模板 {0}{1} 等一致 |
| 文案规范 | 编写新报错 | 对照 3.2 模板 | 符合场景模板与禁用表述要求 |
